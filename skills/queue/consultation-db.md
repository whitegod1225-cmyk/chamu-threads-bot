# consultation-db.md ── 育児相談ナレッジベース

## このファイルの使い方（Claude Codeへ）

ちゃむ。が育児相談メッセージを貼ってきたら、Threads投稿案の作成に加えて、
このファイルの一番下に以下フォーマットで「相談カルテ」を1件追記すること。

追記するだけでよい。過去のカルテを書き換えない。連番（No.）は既存の最大値+1。

**10件溜まるごとに、以下を分析してちゃむ。に報告すること：**
- 悩みの型の分布（食事/睡眠/イヤイヤ/園生活/きょうだい/親のメンタル 等）
- まだ書いていない悩みの型（鉱脈）
- 「定番」と「逆張り」アングルの比率

---

## note連携ルール（重要）

このDBは `/note-article-seller` スキルのネタ元にもなる。以下を必ず守ること：

- 新しいカルテを追記するときは「記事化ステータス：未使用」を必ず付ける
- note記事が完成したら、使ったカルテの「記事化ステータス」を
  「済（記事タイトル／完成日）」に書き換える。完成済みのカルテは二度と提案しない
- ちゃむ。が「カルテからネタ拾って」「在庫から記事書いて」と言った場合、または
  `/note-article-seller` 側でネタが未指定のまま記事生成が呼ばれた場合は、
  このファイル内の「未使用」カルテだけを対象に候補を選定する

## カルテ・フォーマット

```
## 相談カルテ No.〇〇
- 日付：
- 子の月齢/年齢：
- 悩みの型：（食事/睡眠/イヤイヤ/園生活/きょうだい/親のメンタル/その他）
- 相談の核心：（表面の悩みではなく本質を1行で）
- 採用した解決アングル：（定番/逆張り どちらかも明記）
- ボツにしたアングルとその理由：
- 関連商品カテゴリ：（例：ホワイトノイズマシン・抱っこ紐パッド・冷感シーツ等。URLは不要）
- 投稿の反応：（後日追記欄・空欄でOK）
- 転用先候補：（note記事/Kindle章/ここなら台本 のどれに使えるか）
- 記事化ステータス：未使用（note記事にしたら「済（タイトル／完成日）」に書き換える）
```
