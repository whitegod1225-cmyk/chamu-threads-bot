# 05_affiliate.md ── アフィリエイト方針
## 基本方針
- 楽天アフィリエイトをメインに使用
- 「売りたい」感を出さない。実体験ベースで自然に紹介

## 商品カテゴリ（優先順）
1. ベビー用品（おむつ・授乳グッズ・ベビー食器など）
2. 育児グッズ（抱っこ紐・ベビーカー・おもちゃ）
3. 育児と関連できる日用雑貨（収納・洗濯グッズ）
4. 料理雑貨（時短調理グッズ・離乳食関連）

## 投稿フォーマット（固定）
アフィリエイト投稿はすべてこの2段構成で作る。

### 1枚目（本文）：3行以内
- 場面描写・共感・フックのみで構成する
- 商品名・価格・スペックは絶対に書かない
- 「ありきたりな困りごと」→「自分ごと化できる具体的場面」の順で刺す
- **具体的な数字を必ず1つ入れる**：期間（3年・3ヶ月）・回数（毎晩・3回）・変化量（1週間で）など。数字があると読者がリアルさを感じる
- **最終行は必ず中途半端に終わらせる**：続きを読まないと意味が完結しない形で切る。コメント欄を自然に読みたくなる仕掛けにする

【クリフハンガーのバリエーション】「だって」を多用しない。以下から状況に合わせて選ぶ：
- 「その方法が、、、」← 解決策を引っ張る
- 「ひとつ変えただけで、、、」← シンプルな転換
- 「理由を話すと、、、」← 説明への期待感
- 「解決策が意外すぎて、、、」← 驚きへの期待
- 「いい方法が一つあって、、、」← 教えてあげる感
- 「これがすごくて、、、」← 感情で引っ張る
- 「それが」「だって」は使ってよいが連続しないように投稿ごとに変える

【接続詞で文と文を引っ張る】クリフハンガーの簡易版。文末に置くと「まだ何かある」と感じさせ離脱を防ぐ：
- 「ちなみに、」「そして、」「あと、」「でも実は、」
- 例：「ゴキブリは◯で駆除しよう。**ただし、**」← この後に続きを読まずにはいられない
- ⚠️ 毎回同じ誘導文は飽きられる。パターンを3〜4個用意してローテーション

### 情報の小出し3段構成（離脱を防ぐ設計）
最初に全部見せると「もういいや」で離脱される。以下の流れで小出しにする：
- **冒頭**：結論をチラ見せ（全部は見せない）
- **中盤**：詳細を順番に展開
- **終盤**：補足情報をコメント欄へ誘導

「まだ続きがある」と思わせ続けることで最後まで読んでもらう。
本文で完結させると満足して離脱される→コメント欄に「もう少し知りたい」状態で送り込むのが正解。

### コメント欄（セルフリプライ）
- 1枚目のクリフハンガーの続きを冒頭で自然につなげて書く
- **ブランド名・商品名を必ず具体的に書く**（「ブラウン オーラルB」「ムーンスター スーパースター」等）。「これ」「こちら」だけでは信頼感が下がる
- 具体的なベネフィット・使用感を2〜4行で書く
- URLの直前に**CTA（行動誘導文）を1行**入れる → `13_cta-patterns.md` から状況に合うものを選ぶ
- 末尾に「（PR）」を明記（CTAと同じ行に続けてよい）
- 楽天アフィリエイトURLを置く

### キュー管理
- `affiliate-queue.md` で下書き・文体調整を行う
- 仕上がったら **post-queue.md に移動**して自動投稿キューに乗せる
- affiliate-queue.md に残したままでは自動投稿されない

### アフィリエイト投稿の配置ルール（必須）
- **連続投稿は絶対に禁止**：アフィリエイト投稿を連続して並べない
- **1日1本まで**：post-queue.mdでは5投稿ごとに1本を上限とする
- **配置の目安**：通常投稿4〜5本につきアフィリエイト1本を挟む
- post-queue.mdへ追加する際は前後の投稿を確認し、アフィリエイト投稿が隣り合わないように挿入する

## 育児ジャンルの訴求原則

育児グッズは「見た目だけで売れる商品」ではなく「言葉で売る商品」。
画像を並べるだけでは動かない。**読者の悩みを言語化してから商品を差し出す**構成が必須。

訴求の流れ：悩みの代弁 → 共感・自分事化 → 「これで変わった」の転換 → 商品（コメント欄）

## 楽天イベント連動ルール

以下のイベント期間中は成約率が上がるため、イベント前日〜初日に合わせて投稿を優先する。
affiliate-topics.mdの商品候補に「イベント対応」フラグをつけて管理する。

| イベント | 時期（毎月） | 対応 |
|---------|------------|------|
| お買い物マラソン | 月1〜2回（不定期） | 前日〜初日に投稿 |
| 楽天スーパーSALE | 3月・6月・9月・12月 | SALE開始日に投稿集中 |
| 5と0のつく日（ポイント5倍） | 毎月5・10・15・20・25・30日 | 当日朝に投稿 |
| 39ショップの日 | 毎月9・19・29日 | 当日投稿 |

**確認先**：https://calendar.rakuten.co.jp/

## イベント前仕込みルール

イベント当日に慌てて商品を探さないための準備フロー。
**イベント開始3日前までに以下をすべて完了させる。**

### ステップ

```
【3日前】
□ 楽天カレンダーでイベント日程を確認
□ python event-prep.py --event [イベント名] --start [YYYY-MM-DD] を実行
   → 未仕込み候補リスト＋/affiliate-writer 用ブリーフが出力される
□ Claude に貼り付けて投稿文を生成
□ 生成した投稿に <!-- event: [イベント名] --> タグを追加

【前日】
□ post-queue.md に投稿を追記して git push
□ python event-sort.py --event [イベント名] --start [YYYY-MM-DD] --dry-run でプレビュー確認
□ python event-sort.py --event [イベント名] --start [YYYY-MM-DD] で並び替え実行
□ git push（並び替え後のキューを反映）

【当日】
□ GitHub Actions の自動投稿を確認（21:00スロットを優先）
```

### スクリプトの使い方まとめ

| スクリプト | 役割 | コマンド例 |
|-----------|------|----------|
| `event-prep.py` | 候補発掘・ブリーフ出力 | `python event-prep.py --event マラソン --start 2026-07-05` |
| `event-sort.py` | キュー並び替え | `python event-sort.py --event マラソン --start 2026-07-05` |
| `event-sort.py --dry-run` | 並び替えプレビュー | `python event-sort.py --event マラソン --start 2026-07-05 --dry-run` |

### イベントタグ一覧（post-queue.md 投稿ブロック内に追記する）

```markdown
<!-- event: マラソン -->
<!-- event: スーパーSALE -->
<!-- event: 5と0 -->
<!-- event: 値上げ前 -->
```

タグなし投稿は「通常」として扱われ、並び替えの影響を受けない。

### イベント種別×推奨型

| イベント | 使うべき型（第1候補） | 使うべき型（第2候補） | 理由 |
|---------|------------------|------------------|------|
| お買い物マラソン | S-19（値上げ→緊急性型） | S-18（読者共犯型） | 「マラソン中に買わないと損」の体験談損失訴求が刺さる |
| スーパーSALE | S-4（速報興奮型） | S-6（まとめ型） | 大型セールはまとめ型でURL複数を一気に届けると効率的 |
| 5と0のつく日 | S-8（衝動購入報告型） | S-18（読者共犯型） | 「今日ポイント5倍だから買った」という自分の行動報告が自然 |
| 値上げ前 | S-19一択 | — | 値上げ発表直後が最もフックが刺さる。発表翌日には効果半減 |
| 再入荷 | S-4（速報興奮型） | S-17（感情爆発後悔型） | 「やっと復活！」の興奮をリアルタイムで届ける |

### 投稿本数の目安

| イベント規模 | 仕込み本数 |
|------------|----------|
| スーパーSALE（年4回） | 3〜4本（開始日・中日・最終日に分散） |
| お買い物マラソン | 2本（開始日・最終日） |
| 5と0のつく日 | 1本 |
| 値上げ前 | 1本（発表直後に即仕込み） |

### セール期間中の行動ルール

- **スーパーSALE・マラソン中は1日7〜10投稿**（通常の1.5〜2倍に増やす）
- 前日から本気で準備する（当日に慌てない）
- スーパーSALEは**3日前から仕込む**
- 時間がない日は過去のバズ投稿をリポストするだけでもOK
- セール終了後も振り返り分析する（何が刺さったかを記録）

## 楽天ランキング自動収集（product-scraper.py）

### 初回セットアップ（1回だけ実施）

**① 楽天APP_IDを取得する（無料・5分）**
1. https://webservice.rakuten.co.jp/ にアクセス
2. 楽天アカウントでログイン
3. 「新しいアプリを作成」→ アプリ名に「chamu-scraper」と入力
4. 発行された `applicationId` をコピー

**② GitHub Secretsに追加する**
- リポジトリ → Settings → Secrets → Actions → New repository secret
- 名前：`RAKUTEN_APP_ID`
- 値：取得したapplicationId

**③ 動作確認（ローカル）**
```bash
# .env に RAKUTEN_APP_ID=xxx を追加してから
python skills/queue/product-scraper.py --dry-run
```

### 自動実行スケジュール

毎週月曜 9:00 JST に GitHub Actions で自動実行。
結果は `affiliate-topics.md` に候補ステータスで自動追記される。

### 候補追記後の作業（手動・毎週月曜）

```
① affiliate-topics.md を開いて新規候補を確認
② 「投稿アングル」を記入（どんな育児の悩みから入るか）
③ 楽天アフィリエイトポータルで「アフィリエイトURL」を取得して記入
   https://affiliate.rakuten.co.jp/
④ /affiliate-writer で投稿文を生成
⑤ post-queue.md に追加 → git push
```

### フィルタリング基準（product-scraper.py 内の定数）

| 項目 | 基準 | 変更方法 |
|------|------|---------|
| レビュー数 | 100件以上 | `MIN_REVIEW_COUNT` を変更 |
| 評価スコア | 4.0以上 | `MIN_REVIEW_SCORE` を変更 |
| 価格帯 | 1,000〜15,000円 | `MIN_PRICE` / `MAX_PRICE` を変更 |
| 週次追加数 | 最大3件 | `MAX_CANDIDATES` を変更 |

## 注意点
- 「おすすめです！」「ぜひ！」連発はNG
- 「これを買えば解決！」ではなく「うちはこれにした」くらいの温度感
- 単品投稿（型13）・まとめ投稿（型13B）を使い分ける（08_strategy.md参照）

## やりがちな失敗3パターン（クリック率が上がらない原因）

| 失敗 | 何が起きるか |
|------|------------|
| 本文で情報を全部出し切る | コメント欄を見る理由がなくなり離脱される |
| リンクを貼るだけで終わる | 「今押す理由」がなくクリックされない |
| 毎回同じ誘導文 | 読み飽きられてスルーされる |

→ この3つを変えるだけで、同じ閲覧数でもクリック数が変わる

## 冒頭フックのパターン集

1枚目の1行目は以下から選ぶ。**affiliate-examples/ フォルダの実例も参照すること。**
より網羅的なフック集は **`12_hook-patterns.md`** を参照（問題提起型・データ訴求型・共感獲得型など16カテゴリ収録）。

### 基本3パターン

| パターン | 代表フレーズ | 狙い |
|---------|------------|------|
| **断言系** | 「断言します」「ガチで言うと、」 | 信頼感・説得力UP |
| **実績提示系** | 「○○が○○になった」「○○と戦って、やっと勝てた」 | 共感・信頼UP |
| **限定公開系** | 「広まると困るんだけど、」「1日で消します。」 | 希少性・緊急性UP |

【断言系の例】
- 断言しますが、寝かしつけで一番大事なのは○○
- ガチで言うと、スキンケアは高いものより○○が大事

【実績提示系の例】
- 寝かしつけ2時間が15分になった方法があって
- 産後ずっとダサかったけど、これで垢抜けた

【限定公開系の例】
- これ言うと売り切れそうだけど、夜泣き対策で一番効いたやつ
- 広まると困るんだけど、○○に効いたやつ教える

### 追加パターン（affiliate-examplesより抽出）

| パターン | テンプレート | 狙い |
|---------|-----------|------|
| **やめた転換系** | 「〇〇はもうやめた。。。それよりこっちの方が断然良い」 | 認知商品をネガに使って比較・転換 |
| **ターゲット断言系** | 「〇〇が不安な人、これ本当に全員買った方がいい」 | 悩みで絞り込んでから断言 |
| **問いかけ共感系** | 「〇〇って、〇〇じゃない？」 | 読者の体験を代弁・共感で引き込む |
| **自虐転換系** | 「〇〇に憧れて買ったけど、ズボラすぎて卒業しました（笑）」 | 自虐で親近感→代替品へ転換 |
| **リスク数値系** | 「真夏の車内は50度まで上がる」 | 数値でリスクを可視化・危機感を作る |

【コメント欄で使える強い実感ワード】
- 「我が家の救世主」「ストック確定」「毎日ヘビロテ」
- 「去年から使ってる」（継続使用で信頼を出す）
- 「私も使うけど」（複数用途でコスパ感を上げる）
- 「〇〇しなくてもいい」（手間のかかるNG法との対比）

【緊急性・背中を押す一言】
- 「本格的に〇〇になる前には売り切れるから早めに」
- 「1000円ちょっとで対策できる」（コスト感の明示）
- 「ただでさえ〇〇なのに、これ以上増やさなくていい」

### 上級パターン（affiliate-examples 05〜08より）

| パターン | テンプレート | 向いているケース |
|---------|-----------|--------------|
| **速報緊急型** | 「うそでしょ？！〇〇が値上げ？！今ここなら〇〇円！しかも…」 | セール・値上げ前・生産終了 |
| **有益情報導線型** | 知識7割で信頼を積んでから「ちょうどいまセール中！」で誘導 | 保育士知識×楽天セール期間の重なり |
| **購買宣言型** | 「明日半額なるから買う。さすがに〇〇では〇〇できんw」 | 自分の購買体験をリアルタイムで共有 |
| **知らなかった型** | 「〇〇は〇〇週までって言われた。知らなかった、、、いつもここで半額で買ってるよ」 | 意外な育児知識＋購買情報の組み合わせ |
| **まとめ→アフィ導線型** | 「ガチで〇〇を伸ばすなら❶〜❾は必須」→翌日に「昨日のリスト、楽天で買えるやつまとめた↓」 | 有益まとめで保存率を上げてから翌日アフィ |
| **セール攻略ハック型** | 「正直に言う、今夜の勝負。早くチェックできるかで数万円変わる」 | セール前日〜初日に攻略法として届ける |
| **行動介入型** | 「〇〇しようとしてる〇〇ちょっと待って✋今の楽天セールで〇〇が〇〇%OFF」 | 実店舗に行こうとしているターゲットをセール期間中に楽天へ誘導 |
| **第三者証言型** | 「ママ友に教えたら『〇〇で1番いい情報』って感謝された」 | 自分+専門家+第三者の三重で信頼構築 |
| **訳あり切れ端型** | 「形は不揃いだけど品質は職人クオリティそのもの」 | コスパ×品質保証のギャップ訴求（食品・お菓子に有効） |
| **生活日記型** | 「今日の子どもの様子」として書き、商品名を1枚目に1回だけ自然に登場させる。コメント欄は最小限（1〜2行+URL） | 宣伝感ゼロ・信頼感が最大になる。ちゃむ。の育児日記スタイルと最も相性が良い |
| **「ここ一択」条件断言型** | 「〇〇するファミリーはここ一択！！！そもそも〇〇なのにセールで更に」 | 条件を設定して迷いをなくす＋二重のお得感 |
| **「バグ」コスパリアクション型** | 「えっ、バグってる？〇〇がこの価格って？？？」コメント欄も1行のみ | ブランド品・大幅割引時の超短いフック |
| **URLまとめキュレーター型** | 「❶ 〇〇 50%OFF → URL ❷ 〇〇 30%OFF → URL」（説明なし） | セール期間中に複数商品を一気に届ける。制作コスト最小 |
| **雑談→商品転換型** | 「おつかれ〜🙌今日も暑かったね。そういえば今夜からセール↓」 | 日常会話で温度感を作ってから商品へ自然につなげる |

【コメント欄が超短くてよいケース】
- 購買宣言型・知らなかった型：「前開きで授乳できるのもいいよね！↓コレだよ↓」程度で十分
- 読者がすでに「欲しい」状態になっていればURLだけで動く

## 「保育士が現場で使って自宅でも買った」型（★バズ実績あり）

参考投稿：@pacyu10 ハサミ投稿（61.6K views）→ `09_references.md` アフィリエイト参考投稿⑨

**構造の核心**：「おすすめです」という宣言ではなく「自分で全部買い替えた」という行動の事実を届ける。専門家が自腹で選んだという情報量だけで購買意欲が動く。

### 使うべき状況
- ちゃむ。が保育士として現場で実際に使った経験がある商品
- 「何十種類も試した末に行き着いた」と言えるカテゴリ（はさみ・粘着テープ・収納グッズ等）

### ちゃむ。軸への変換テンプレート

| 原型 | ちゃむ。バージョン |
|------|----------------|
| 「保育士として現場で長年使ってきた」 | 「保育士として現場でずっと使ってた〇〇、自宅用にも同じの買った」 |
| 「自宅を全部そのブランドに統一した」 | 「子ども3人ぶん揃えた」「うちもこれにした」 |
| ストレスがゼロになった | 「〇〇がなくなった」「毎回イライラしてたのに一切なくなった」 |

### 注意点
- **「保育士歴16年だからわかる」とは書かない**（自分の権威を宣言するのではなく、行動の事実を届けることで権威が自然に出る）
- 「おすすめ」「ぜひ」は使わない。「うちはこれにした」「同じの買った」で十分
- 高views低いいね比率（保存＆スルー型）になりやすいカテゴリで特に有効

## 機能ではなくベネフィットで語る

商品説明は「スペック（何ができるか）」ではなく「ベネフィット（どう変わるか）」に変換する。
人はスペックではなく、それを使って得られる**未来**を買う。

| NG（機能） | OK（ベネフィット） |
|-----------|----------------|
| 連続使用時間8時間 | 途中で止まらず、1日中使える |
| 軽量コンパクト設計 | カバンに入れっぱなしでも存在忘れるレベル |
| お手入れ簡単 | 食洗機にポイで終わり。洗い物増えん |

## バズ投稿の再利用ルール

- 伸びた投稿は**3日後に同じ内容で再投稿**する（アルゴリズム上有効）
- 同じ商品は切り口を変えて**最低3回**使い回す
- バズ投稿・アフィ投稿は**20〜21時台**に配置する（スマホ利用ピーク）

## 楽天アフィ固有の数字

- 狙う料率：**10%以上**の商品を優先
- クリックの有効期間：**89日間**（クリックから89日以内の購入が対象）
- 報酬画面の確認：週1でOK（毎日見るとメンタルが削れる）

## 週次成果レビュールール

**毎週月曜**に楽天アフィリエイト管理画面を確認して `affiliate-history.md` に記録する。

| チェック項目 | 判断基準 | 次のアクション |
|------------|---------|-------------|
| クリック数が多いのに成約ゼロ | コメント欄のベネフィット説明が弱い | コメント欄をリライトして再投稿候補に |
| クリック数自体が少ない | 本文フックが弱い or 配置タイミングが悪い | フックを変えて別商品で類似投稿を試す |
| 成約率が高い商品カテゴリ | 読者ニーズと合致している | 同カテゴリで追加投稿候補をaffiliate-topics.mdに追加 |
| イベント期間中にクリック急増 | セール連動の効果が出ている | 次回イベントでも同商品・同カテゴリを優先 |

## note導線設計

### 基本導線
Threads投稿 → 無料note（集客装置）→ 有料note（購入）

直接有料noteへ誘導する「直通ルート」はNG。信頼がない状態では購入されない。

### 無料noteの役割
- 購入ハードルをゼロにする（タダだから読まれる → 価値を感じる → 続きが欲しくなる）
- 信頼を先払いする（読後に「この人は本物だ」と感じさせる）
- ターゲットを自動選別する（本気で興味がある人だけが無料noteまで読む）

### 無料/有料の境界線3基準
**基準1：問題の理解は無料、問題の解決は有料**
- 無料note：「なぜ自分はうまくいかないのか」を気づかせる内容
- 有料note：「どうすればうまくいくのか」の具体的な手順・解決策

**基準2：読者の言葉は無料、自分の言葉は有料**
- 無料note：読者が日常で使う言葉・感じる感情・経験する状況で語る
- 有料note：ちゃむ。独自の視点・保育士16年でしか語れない知識・オリジナルの言語化

**基準3：1テーマを深く・800〜1,200文字**
- 無料noteで扱う問題は必ず1つに絞る
- 浅く広く書いた無料noteは「なるほど感」を生まない
- 深く狭く書いた無料noteが「この人は本物だ」という信頼を生む

### 無料noteの設計手順
1. 有料noteが解決する「最終的な問題」を1行で書く
2. その最終問題の「前段階にある問題」を抽出する（これが無料noteのテーマ）
3. 前段階の問題を読者の言葉で深く掘り下げる
4. 解決策は書かない
5. 末尾に7ステップCTAを置く（→ 11_monetize-prompts.md のプロンプト13を使う）
