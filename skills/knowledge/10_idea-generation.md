# 10_idea-generation.md ── バズテーマ独自案生成プロンプト集

## 概要
既存のバズテーマを分析・解体し、パクリに見えない独自テーマを生成するための7プロンプト。
`/writer` や `/researcher` が新テーマを作る際に使う。

`[テーマ]` の部分に参考にするバズ投稿のタイトルや内容を入れて使う。

---

## ① 3軸バズ解剖プロンプト
感情軸・欲求軸・文脈軸で分解し、ラベルとフレームだけを刷新する。

```
「[テーマ]」がバズった理由を「感情軸・欲求軸・文脈軸」の3つで分解して。
その後、各軸のヒット要因は完全に残しながら、ラベルとフレームだけを刷新し、
一見全く別テーマに見える独自案を5つ作って。
各案に「元テーマとの差別化ポイント」と「なぜ刺さるか」を一言で添えること。
```

---

## ② 感情トリガー移植プロンプト
感情トリガーを維持したまま、切り口・立場・対象読者のどれかを入れ替える。

```
「[テーマ]」の核心にある読者の感情トリガーを特定して。
次に、そのトリガーを維持したまま、切り口・ラベル・語り手の立場・対象読者の
どれか一つ以上を入れ替えて、全く新しいテーマに見える案を5つ提案して。
各案に変換した要素と差別化の根拠を明記すること。
```

---

## ③ バズ骨格フレーム換装プロンプト
共感・驚き・有用性のどの要素が刺さっているか分析し、骨格は保ちつつ表現を刷新する。

```
「[テーマ]」を参考に、バズ構造（共感・驚き・有用性）のどの要素が刺さっているか分析して。
その構造を骨格として保ちつつ、フレームと表現ラベルだけを大胆に入れ替えた
独自テーマ5案を出して。
「元と何が違うか」「どんな読者に刺さるか」を各案に添えること。
```

---

## ④ 3手法ラベル変換プロンプト
①ラベル変換 ②視点移植 ③対象シフト の3手法を組み合わせる。

```
「[テーマ]」のヒット軸を特定し、①ラベル変換②視点移植③対象シフトの3手法を使って
独自テーマを5案作って。
各案にどの手法を使ったか・なぜ新しく見えるか・刺さる感情は何かを簡潔に添えること。
元テーマと被らない独自性を最優先すること。
```

---

## ⑤ 訴求本質リフレームプロンプト
「読者が本当に求めているもの」を定義し、フレームだけを全て入れ替える。

```
「[テーマ]」が持つ訴求の本質（読者が本当に求めているもの）を一言で定義して。
その本質は変えずに、フレーム・文脈・語り口・ラベルを全て入れ替えた新テーマ5案を提案して。
各案に「訴求の本質との繋がり」と「独自に見える理由」を添えること。
```

---

## ⑥ 3要素バズ因数分解プロンプト
感情・情報・社会的証明で分解し、業界・立場・時間軸・規模感のどれかを入れ替える。

```
「[テーマ]」のバズ要因を感情・情報・社会的証明の観点で分解して。
その要因を保ちながら、業界・立場・時間軸・規模感のどれかを入れ替えてフレームを刷新し、
独自テーマ5案を生成して。
なぜパクリに見えないか・なぜ刺さるかを各案に必ず明記すること。
```

---

## ⑦ フレーム移植・別物化プロンプト
成功フレームが与える認知・感情・行動変容を特定し、全く異なるラベルに移植する。

```
「[テーマ]」を"成功しているフレーム"として分析し、そのフレームが読者に与えている
認知・感情・行動変容を特定して。
次に全く異なるラベルと文脈にそのフレームを移植し、別物に見えながら同等の訴求力を持つ
独自テーマを5案作って。各案に移植した要素と期待される反応を添えること。
```

---

## 使い方ガイド

| プロンプト | 向いている場面 |
|-----------|--------------|
| ① 3軸バズ解剖 | バズ理由が明確な投稿を徹底分解したい時 |
| ② 感情トリガー移植 | 感情共感系（型A・F）の新テーマを作る時 |
| ③ バズ骨格フレーム換装 | 有用性系（型3・8）の構成を流用したい時 |
| ④ 3手法ラベル変換 | 手法を意識しながら体系的に発散したい時 |
| ⑤ 訴求本質リフレーム | 「何が刺さるか」の本質から逆算したい時 |
| ⑥ 3要素因数分解 | 社会的証明（保育士16年）を活かしたい時 |
| ⑦ フレーム移植 | 他ジャンルのバズ構造をそのまま借用する時 |
