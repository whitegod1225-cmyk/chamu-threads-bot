# 22_algorithm-adaptation.md ── アルゴリズム変化検知・適応プロトコル
作成：2026-07-07

---

## ■ 基本思想

Threadsのアルゴリズムは告知なく変化する。
変化を「数字の変化」として検知し、「投稿設計の変化」で応答する仕組みをここで定義する。

**原則：アルゴリズムが変わっても「読者の感情を動かす」本質は変わらない。**
変えるのは「どのエンゲージメントを最大化するか」の重み付けだけ。

---

## ■ 現在の推定スコア式（2026年7月時点）

```
score = likes × 3 + comments × 5 + reposts × 2
```

この式の重みが変化することが「アルゴリズム変更」の実態。

---

## ■ アルゴリズム変化の検知指標

### 週次モニタリング（/supervisorが実行）

毎週月曜に以下を計算してsupervisor-report.mdに記録する：

```
週次アルゴリズム健全性スコア
━━━━━━━━━━━━━━━━━━━
① 直近7投稿の平均いいね率  = ___% （基準: 0.3%以上）
② 直近7投稿の平均コメント率 = ___% （基準: 0.05%以上）
③ 直近7投稿の平均質スコア  = ___% （基準: 1.0%以上）
④ 同型投稿の前週比スコア変化率 = ___% （±20%以内が正常）
```

### アルゴリズム変更の疑いフラグ

以下のどれかが2週間連続して発生したら「アルゴリズム変更の疑い」として記録する：

```
🚨 FLAG-1: 同型投稿のスコアが前月比 -40% 以上低下
🚨 FLAG-2: いいね率は横ばいなのにコメント率が +100% 以上増加（コメント重視へ移行）
🚨 FLAG-3: B型知識投稿のviewsが突然+200%（リーチアルゴリズム変化）
🚨 FLAG-4: 型2の質スコアが B型知識の質スコアを下回った（共感より情報が評価される変化）
```

## ■ アルゴリズム変化パターンと対応戦略

### パターン1：コメント重視強化期（現在に近い状態）

**兆候**：
- 型2投稿のスコアが知識系を大きく上回っている
- 「うちだけ？」「同じ人いる？」型の末尾でコメントが集中する
- 問いかけ型（低views）のスコアが相対的に上昇

**対応策**：
- 型2（敗因報告型）を週2本以上維持
- 全投稿の末尾に「余白のある問いかけ」を必ず入れる
- コメント欄の続きを意識的に「答えやすい質問」で終わらせる

---

### パターン2：いいね重視期

**兆候**：
- 感情系（代弁型）のいいね率が突然上昇
- 型2の相対スコアが下がり、E型（炎上覚悟共感型）が上昇
- 「保存」系コンテンツのスコアが下がる

**対応策**：
- フックの感情訴求を強化（「夜9時、寝顔見ながら〜」型の場面描写）
- 告白型（「正直に言うね」）の頻度を上げる
- 概念再定義型の比率を上げる（読者の代弁機能が強い）

---

### パターン3：リーチ拡大期（新規フォロワー獲得優先）

**兆候**：
- B型知識投稿のviewsが突然 2〜3倍に伸びる
- フォロワー外へのリーチが増える（新規フォロワーが増加）
- 保存・リポストが相対的に高くなる

**対応策**：
- B型知識投稿の比率を一時的に上げる（週2本 → 週3本）
- 「保育士が現場で使ってた〇〇」型の具体的ノウハウを増やす
- リポストされやすいリスト型・比較型を増やす

---

### パターン4：シェア・外部拡散重視期

**兆候**：
- B型知識のreposts数が突然上昇
- 「保存して使いたい」型コンテンツのスコアが上昇
- コメントより reposts の重みが高くなる

**対応策**：
- 「まとめてくれた」「誰かに送りたい」コンテンツを増やす
- 7選・5選・比較表型の投稿を定期的に入れる
- アフィリエイト投稿との連携強化（商品情報の「誰かに教えたい」動機）

## ■ 投稿型比率の標準設定と変更ルール

### 現在の推奨比率（コメント重視期）

```
型2（敗因報告・共感本音）: 30%  ← コメント誘引の主力
B型（知識有益系）:        25%  ← リーチ獲得・信頼構築
概念再定義型:            20%  ← いいね誘引・罪悪感溶かし
問いかけ・あるある型:    15%  ← コミュニティ形成
型A（自己紹介）:         10%  ← 定期的に入れる（10投稿に1本ルール）
```

### 比率変更の判断ルール

```
FLAG発生 → 2週間様子見 → FLAG継続ならその週から比率変更
変更幅: 各カテゴリ最大±10%（急激な変更はアカウントの一貫性を損なう）
変更記録: supervisor-report.mdに変更日・理由・新比率を必ず記録
```

## ■ アルゴリズム変化を「数字の前に感覚で気づく」サイン

> **以下のセクションはFable 5による感覚的な変化検知の記述（2026-07-07）**

**数字より先に感覚で検知するサイン（3つ）**

1. **同精度の投稿でviews/likesの比率が崩れる**：いつもの型2でviewsだけ落ちた→初速シグナル（いいね）の重みが下がった可能性。逆にreplies少ない投稿のviewsが伸び始めた→コメント重視が緩んだサイン

2. **コメント欄の質の変化**：「わかる」の一言コメントが減り、長文体験談だけが残る→短い反応の価値が下がり、会話の深さが評価され始めた兆候

3. **古い投稿の再浮上**：投稿後48時間を過ぎたB型が再びviewsを稼ぎ出す→保存・検索性の評価が上がったサイン。この時期は型2（鮮度依存）の比率を下げ、B型の在庫価値を上げる

**アルゴリズム変化への応答原則**

アルゴリズムが重視する指標＝設計すべき「読後感情」が変わる：

| 重視指標 | 最大化する感情 | 着地の設計 |
|---------|--------------|-----------|
| いいね | 完結した共感（代わりに言ってくれた） | 完璧な代弁で締める。余白を残さない |
| コメント | 未完結の共感（私も言いたい） | 代弁を8割で止め、読者の物語の呼び水にする |
| 保存・シェア | 実用＋他者共有欲 | B型寄り。「明日使える一手」「パートナーに読ませたい免責」を含める |

移行時に変わるのは「どこで止めるか」だけ。型2とB型は感情の完結地点が違う2つのレバーであり、比率調整だけで対応する。新しい型をゼロから作る必要はない。

## ■ 長期的な変化耐性の設計思想

アルゴリズムが何に変わっても機能し続ける投稿の条件：

1. **読者の感情が動く**：どの指標が重視されても、感情が動く投稿は何らかの行動を引き出す
2. **ちゃむ。にしか言えない**：保育士×母の二重視点は代替不可能な差別化
3. **読者の言えない本音を代弁する**：「これを言ってくれる人を離したくない」フォロー動機になる
4. **情報と感情の適切なバランス**：純粋な知識は拡散されるが共鳴は生まない。感情だけでは信頼が薄い

**この4条件を満たす投稿は、アルゴリズム変化に対して最も耐性が高い。**
