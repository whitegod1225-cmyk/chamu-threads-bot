## ■ スコア計算式（実測から逆算）

```
score = likes × 3 + comments × 5 + reposts × 2
```

**重要な発見**：コメント(5)がいいね(3)よりも高重みで計算される。
→ Threadsアルゴリズムは「コメントを引き出す投稿」を最も評価している。
→ 高viewsでも「知識を届けるだけで完結する投稿」はスコアが伸びない。

## ■ エンゲージメント種別と投稿型マッピング

```
いいね  = 「代わりに言ってくれた」「わかる」感情の表明行動
         → 読者の言えなかった本音を代弁した時に来る
         → 型2（敗因報告）・概念再定義型が最も引き出しやすい

コメント = 「自分も言いたい」「経験を話したい」余白への反応
         → 問いかけ・不完全な告白・「うちだけ？」が引き金
         → 型2のうち「朝の戦争」「呼び出し」は63/59コメントを獲得

リポスト = 「誰かに見せたい情報密度」
         → 具体的なノウハウ・保育士視点の発見・数字入りリスト
         → B型知識（パパとイヤイヤ：16reposts）で出やすい

保存    = 「後で使える・見直したい」実用性
         → NG→OK変換型・ハック集・チェックリスト
         → Threads APIでは現在取得不可だがscore外で効く
```

### 型2（敗因報告型）の感情メカニズム

読者（30-35歳・育児中ワーママ）は、日中「自分は子育てがうまくいっていない」という薄い罪悪感を常に抱えてスクロールしている。そこに「今日の保育士としての敗因、報告します」という1行が入る。

ここで起きるのは二段階の感情反転：

1. **「保育士なのに負けるの？」という権威の自己崩壊**。プロが「敗因」を「報告」する。この落差がスクロールを止める。読者の中で「専門家=正解を知っている人」というポジションが自ら崩れる瞬間を目撃する期待が生まれる。
2. **本文で「わたしと同じだ」に着地**。プロですら負ける相手（イヤイヤ、朝の支度、呼び出し）に自分も負けている。読者の罪悪感が「負けて当然」という**赦し**に変換される。

この型の本質は「**面白い話を読んだ**」ではなく「**今日の自分が赦された**」こと。だから感謝に近い感情でいいねが押される。

さらに「報告します」という語尾は**軍隊報告のパロディ形式**で、読者に「私の敗因も報告します」という返信フォーマットを暗黙に配布している。「朝の戦争」(59replies)「呼び出し」(63replies)でコメントが伸びたのは、テーマが**全読者に今日の実体験がある**もの（朝・呼び出し）だったから。

**いいね vs コメントの感情原理**

- **いいね** = 「代わりに言ってくれた」への感謝・連帯署名。読者の中で感情が**完結している**時に押される。
- **コメント** = 「私にも言いたいことがある」。感情が完結せず、**自分の物語が喚起されて溢れた**時に発生する。投稿が読者の体験の「呼び水」になっている状態。

型2の926likes(敗因報告)は完成度の高い代弁。59-63repliesの投稿は「あなたの番です」という余白の設計差。

**型2が機能する必要条件**
- 権威（保育士/プロ）が自ら負ける構図があること。素人の失敗談ではダメ
- 敗因の相手が「読者全員が今日も戦っている相手」であること（朝・食事・寝かしつけ等）
- 「報告」形式など、読者が自分の話を返信できる**フォーマットの配布**があること

### B型（知識・有益系）の感情メカニズムと限界

「パパがいるとイヤイヤが激化する仕組み」(345,945views / 453likes / 8replies)。この投稿で読者に起きるのは共感ではなく「**謎が解けた**」という認知的快感。

- 高リーチの理由：「あるあるだけど理由を知らない現象×答えの約束」はアルゴリズムに乗りやすい（滞在時間・タップが伸びる）。
- 低エンゲージの理由：**知識は読者の感情を動かさない。読者の頭を動かすだけ**。「なるほど」で感情が完結し、しかも「なるほど」は署名（いいね）する動機が弱い。知識は受け取った瞬間に**読者の所有物になる**ため、発信者への感謝が発生しにくい。「教えてもらった」より「知った」という自己完結で終わる。

**345,945views→453likesの解剖**：いいね率0.13%。これは「読まれたが、読者の物語に触れなかった」証拠。イヤイヤ激化の仕組みは「パパの話」であり、読者（ママ）自身の傷や努力には触れていない。**共感は「私の話をされた」時にしか起きない**。B型は「子どもの話・夫の話」をしており、主語が読者からズレている。

**B型が機能する必要条件**
- 現象が「ほぼ全家庭で今週起きたこと」レベルに普遍的であること
- 「親のせいではない」という免責が構造に含まれること
- 1行目が「現象の名指し」になっていること（読者が「うちだ」と3語以内で判定できる）

### 読者感情移動マップ：型2（敗因報告型）

```
[状態0] 夕方〜夜、育児の疲労と「今日も怒っちゃった」という薄い罪悪感を
        抱えて無目的スクロール中。誰にも今日の苦労を報告していない。

[状態1] 「今日の〜、報告します」で停止。
        感情：軽い笑い＋「プロが負けた？」という好奇心。
        脳内予測：「オチのある失敗談が来る」→安心して読める（説教が来ない保証）。

[状態2] 具体的な敗戦描写を読む。
        感情：「うちも全く同じ」の連続ヒット。1行ごとに自分の今日が再生される。
        重要：ここで描写がプロ固有すぎると「別世界の話」になり離脱する。
        家庭でも起きるディテールに揃えること。

[状態3] 着地点：「プロでもこれ」「明日も負けます」等の自己受容。
        起きる感情：赦し。「負けてる私は普通だった」。
        罪悪感 → 連帯感への変換がここで完了する。
        ※説教・学びに転じたら型2は死ぬ。負けたまま終わること。

[状態4] 行動分岐：
        ・投稿の代弁が完璧 → いいね（感謝の署名）
        ・自分の「今日の戦況」が喚起された → コメント（我が家の敗因報告）
        コメントを狙うなら状態3で「みなさんの敗因も聞かせて」と言わずに、
        報告フォーマット自体を返信テンプレとして機能させる。
        保存・リポストはほぼ来ない（後で読み返す実用性がないため）。
```

### 読者感情移動マップ：B型（知識有益系）

```
[状態0] 同上の無目的スクロール。ただし「子育ての正解を探したい」欲求も常駐。

[状態1] 現象の名指し（例:「パパがいるとイヤイヤが激化する」）で停止。
        感情：「それ！うちも！なんで！？」という未解決の謎の再燃。
        条件：読者が過去1週間に体験した現象であること。

[状態2] 仕組みの説明を読む。
        感情：認知的快感（点と点がつながる）。感情ではなく理解が進む。
        重要：ここに「ママのせいじゃない」という免責を必ず織り込む。
        免責がないと「私が責められている」に転じて離脱・防御反応。

[状態3] 着地点：「だから〜でいいんです」という許可、または明日使える一手。
        起きる感情：「気づかせてくれた感」＋安堵。
        ※共感の温度は上がらない。上げたいなら最後の1-2行で
        「わたしも夫にイラッとしてました」等、ちゃむ。自身の実体験を差す。

[状態4] 行動分岐：
        ・純粋な知識のまま終わる → 保存 or 無反応（高view低like）
        ・免責＋許可で終わる → いいね（安堵の署名）
        ・「夫に見せたい」内容 → シェア/リポスト
        コメントは構造的にほぼ来ない（読者に語るべき物語がない）。
        コメントが欲しい投稿にB型を使わないこと。
```

### 高リーチ低共鳴パターンの解剖

> 「パパがいるとイヤイヤが激化する仕組み」345,945views / 453likes の矛盾

345kビューに対していいね率0.13%。質スコア0.42%。これは「広く読まれたが読者の物語に触れなかった」投稿の典型パターン。

**なぜリーチが高いのか**：「あるあるだけど理由を知らない現象」はアルゴリズムに乗りやすい。1行目「パパがいるとイヤイヤが激化する」は多くの家庭で今週体験した現象の名指しであり、滞在時間が伸びる。

**なぜ共鳴が低いのか**：主語が「パパ・子ども」であり、投稿を読んでいる読者（ママ）自身の感情・努力・苦しさに触れていない。知識は受け取った瞬間に読者の所有物になるため、発信者への感謝が発生しにくい。B型の根本的な限界がここにある。

**対処法**：B型の末尾にちゃむ。自身の実体験1〜2行を必ず入れる（「わたしも夫にイラッとしてました」等）。知識の完結性を少し崩し、ちゃむ。の感情が読者の感情と接触する接点を作る。

## ■ 投稿品質ルーブリック（Sonnet自己採点用）

投稿完成後に以下の5項目を採点。合計15点以上を投稿可とする。

### 1. 1行目引力スコア（1-5点）

- **5点**：「権威の自己崩壊」or「未解決あるあるの名指し」があり、続きの予測が1つに絞れない。かつ読者が0.5秒で「自分の話だ」と判定できる。例：「今日の朝の戦争、報告します」
- **3点**：テーマは伝わるが感情の落差がない。「〜について話します」型。読むかどうかは読者のその日の関心次第。
- **1点**：主語が子ども・専門用語で始まる。読者が自分との関係を判定できない。例：「歯医者さんに連れていくよが逆効果な理由」（状況限定＋NG指摘の予告）

### 2. 感情着地強度（1-5点）

着地で残る感情を必ず1つ特定してから採点する：
- **代わりに言ってくれた感**（型2向き・いいねを生む）
- **気づかせてくれた感**（B型向き・保存を生む）
- **仲間がいる感**（型2向き・コメントを生む）

採点：
- **5点**：上記のどれか1つが明確に残り、かつ「読者への免責 or 赦し」が最終2行にある
- **3点**：感情は動くが「で、私はどうすれば」が宙に浮く。または感情と知識が混ざって着地がぼやける
- **1点**：着地が説教・正解提示で終わり、読者が採点された感覚を持つ

### 3. コメント誘引設計（1-5点）

- **5点**：読者が自分の体験を書き込める「返信フォーマット」が投稿構造に埋まっている（報告形式・穴埋め可能な型）＋テーマが全読者に今日の実体験がある
- **3点**：共感はされるが投稿内で話が完結しており、読者が付け足す余白がない（完璧な代弁＝いいね止まり）
- **1点**：読者に語る物語がない知識完結型、または反論したくなる断定がある

### 4. ちゃむ。固有性（1-5点）

- **5点**：「保育園ではできるのに家ではできない私」等、保育士と母の**矛盾・落差**が投稿の核になっている。この落差はちゃむ。にしか書けない
- **3点**：「保育士です」の署名的言及はあるが、本文はどの育児アカウントでも書ける
- **1点**：肩書なしで成立する一般育児知識。検索すれば出てくる内容

**合格ライン：合計16/20点。コメント狙い投稿は項目3が4点以上必須、リーチ狙い投稿は項目1が4点以上必須。**

### 5. アルゴリズム適合性（1-5点）

```
5点：コメントを誘引する余白がある＋リポスト動機になる情報がある
4点：どちらか一方がある
3点：いいねは来そうだがコメントもリポストも来にくい構造
2点：読者の感情に訴えかけているが行動につながる仕掛けがない
1点：情報伝達のみ。読者の感情が動かない
```

## ■ 投稿パフォーマンス分類

analysis-latest.mdのデータを以下4象限で分類して読む：

```
                高コメント率
                    │
  [C] 問いかけ成功型  │  [A] 最強投稿型
  views低いが会話が   │  型2・場面描写系
  生まれている        │  いいね＋コメント両立
                    │
────────────────────┼────────────────────
  低いいね率          │          高いいね率
                    │
  [D] 空振り型        │  [B] 拡散型
  改善が必要な        │  B型知識系
  投稿               │  リーチ高いが共鳴薄い
                    │
                低コメント率
```

**[A]最強投稿型** を増やすことがスコア最大化の本筋。
**[B]拡散型** はリーチ獲得・アルゴリズム学習には貢献するが、コメント率改善が課題。
**[C]問いかけ成功型** はコミュニティ形成には有効。アカウント初期は重要。
**[D]空振り型** は1行目かテーマ選定を見直す。
