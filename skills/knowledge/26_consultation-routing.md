# 26_consultation-routing.md ── 相談→投稿ルーティングシステム

> 育児相談1件から投稿3本を作る際の、「3本目の枠」をどこに向けるかの判定ルール。
> 相談が来たとき、このファイルを参照してルーティングを決定してから投稿生成に入る。

---

## 投稿生成時の視点ルール

相談文をもとに投稿①②③を作るとき、**「相談を読んだ」視点で書かない。**
「保育士のころに受けた相談・見てきた場面」として読み替えて執筆する。

- ❌「こういう相談をもらいました」「DMで来た悩みです」
- ✅「保育士のころ、よく聞いた悩みがあって」
- ✅「園で何度も見てきた場面だった」
- ✅「16年の現場でこういう保護者と話したことが何度もある」

詳細は `03_writing.md`「相談ベース投稿の視点ルール」を参照。

---

## 基本フロー

```
相談が来る
  ↓
【判定】商品で解決できるか？
  ├─ YES → 投稿③ = アフィリエイト投稿（/affiliate-writer）
  └─ NO  → 投稿③ = note誘導投稿（/note-article-seller → 誘導投稿生成）
  ↓
投稿①②③を post-queue.md に追加
```

## 3本の役割分担

| 枠 | 型 | 狙うエンゲージメント | スキル |
|---|---|-----------------|------|
| **投稿①** | 共感タイプ（型2 告白・報告型 or 概念再定義型） | いいね | /writer |
| **投稿②** | 有益情報タイプ（保育士知識・発達段階） | 保存 | /writer |
| **投稿③** | ルーティング枠 | クリック・購入・note読了 | 下記判定 |

---

## 投稿③のルーティング判定

### 商品で解決できる → アフィリエイト投稿

**条件**：具体的なモノが悩みの解決に直接つながる

| テーマ例 | 商品の方向 |
|---------|----------|
| 寝つきが悪い・夜泣き | ホワイトノイズマシン・おやすみルーティンググッズ |
| 肌荒れ・乾燥 | ベビーローション・低刺激肌着 |
| 離乳食が進まない | 食器・スプーン・ベビーフード |
| おもちゃに飽きる | 知育玩具・木のおもちゃ |
| 保育園グッズ | お名前スタンプ・連絡帳ケース等 |
| 抱っこ・移動が大変 | 抱っこ紐・ベビーカーアクセサリー |
| 入浴が大変 | ベビーバス・沐浴グッズ |
| 衣類の名前付け・収納 | ランドリー用品・収納グッズ |

**実行**：`/affiliate-writer [楽天URL]`
→ URLが未取得の場合は「楽天で検索して取得してください」と案内してから投稿生成に入る

---

### 商品で解決できない → note誘導投稿

**条件**：感情・関係性・認知・考え方が解決の核になる

| テーマ例 | 記事の方向 |
|---------|----------|
| 子どもに怒鳴ってしまう・自己嫌悪 | 怒鳴り後の立て直し方（パターンA） |
| ひとりっ子への後悔・罪悪感 | 後悔の正体を代弁（パターンC） |
| 夫が育児に非協力・温度差 | 夫婦の対話設計（パターンC） |
| 保育園トラブル・先生への不満 | 先生との関係性の見方（パターンA） |
| 発達不安・グレーゾーン | 保育士が見てきた「個性」の話（パターンB） |
| 寝かしつけ消耗・毎晩しんどい | ルーティン見直しの考え方（パターンB） |
| 孤育て・しんどい・誰にも言えない | 孤独感の代弁（パターンC） |
| 子どもへの接し方・言葉かけ | 言い換えスクリプト（型3） |

**実行手順**：
1. `/note-article-seller [相談内容]` を実行して記事を先に作る
2. 記事が完成したら誘導投稿（Threads）を生成
3. 誘導投稿が「投稿③」になる
4. note公開後にURLを誘導投稿のコメント欄に挿入

---

## 判定が迷うときのルール

**両方あてはまる場合**：感情が強い方を優先する
- 「寝つき改善グッズ＋消耗している感情」→ 感情が強ければnote
- 「肌荒れグッズ＋罪悪感」→ 罪悪感が強ければnote

**商品は思いつくが、相談の根本が感情系の場合**：noteに振る
- 商品は一時的な解決。感情の整理の方が読者に長く残る

**商品が楽天で見つからない場合**：noteに振る
- 無理に商品を当てはめない

---

## note誘導投稿の形式

note記事作成後、`/note-article-seller` の出力からThreads誘導投稿を使う。

**形式ルール**（ステップ6出力参照）：
- 本文3〜5行（記事の核心を一言で）
- コメント欄末尾に `📝 → [noteURL]`
- URLが未公開の場合は `[noteURL]` のままキューに追加し、公開後に差し替え

---

## 配置ルール（post-queue.md追加時）

投稿①②③の順番は以下を守る：

```
投稿①（共感）→ 投稿②（有益情報）→ 投稿③（アフィリ or note誘導）
```

ただし直前のキューの状況によって：
- アフィリが連続になる場合 → ①②を先に追加し、③は3〜5投稿後ろにずらす
- note誘導が直前にある場合 → 同じ記事への二重誘導は避ける（別記事ならOK）

```

```
