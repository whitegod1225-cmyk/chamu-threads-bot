## 1. 育児noteの「読者の旅路」設計

読者は記事を「情報を取りに来ている」のではなく「自分の状況が救われるかを確かめに来ている」。
この違いを前提に全章を設計する。

### 3フェーズの感情移動

| フェーズ | 章の位置 | 読者の感情 | 機能 |
|---------|---------|----------|------|
| **認識フェーズ** | はじめに・第1章 | 孤独→共感→安堵 | 「この記事は私のために書かれている」と思わせる |
| **揺さぶりフェーズ** | 第2章・第3章前半 | 問題の解像度が上がる→焦り→発見 | 「自分がなぜうまくいかないか」が腑に落ちる |
| **転換フェーズ** | 第3章後半〜おわりに（有料核心） | 希望→具体的行動意欲→信頼 | 「今日から変えられる」を確信させる |

### 感情が動かない記事のNG例

- 全章が「情報の列挙」になっている（知識は入るが感情が動かない）
- 「なぜそうなるか」で終わって「私はどうすればいいか」が来ない
- 共感パートが短すぎて「理解してくれる人」として認識されていない
- 有料部分の核心が「さらに詳しい知識」だけで「感情の転換」がない

## 2. 各章ごとの感情設計テンプレート

記事を書く前に、各章の「狙う感情」を先に決める（感情ファースト・内容セカンド）。

### はじめに（無料）

**狙う感情**：孤独感の解消 → 「わかってくれる人がいた」という安堵

**設計のポイント**：
- 読者が感じているモヤモヤを、読者より先に言語化して「あなたのことを見てた」と伝える
- 「完璧な保育士の答え」ではなく「私も同じだった」から入る
- ちゃむ。の実体験の感情実況を必ず入れる（「しんどかった」「情けなかった」など）

**NG**：「この記事ではXXXについて解説します」という案内から始める

---

### 第1章：よくある誤解を崩す（無料）

**狙う感情**：「そういうことだったのか」という認知的快感 + 軽い罪悪感の解放

**設計のポイント**：
- 「多くのママがやりがちな誤解」を1つ取り上げて崩す
- 崩す前に「その誤解をやってしまう気持ちは当然」と免責を入れる
- 「間違ってた」ではなく「知らなかっただけ」で着地させる

**免責の書き方**：
```
悪い例：「〇〇はNGです」（罪悪感→防御反応→離脱）
良い例：「〇〇したくなる気持ち、すごくわかります。でも実は——」（共感→受容→好奇心）
```

---

### 第2章：現場のエピソード・保育士視点（無料〜有料境界付近）

**狙う感情**：「この人は本物だ」という信頼 + 「私の子にも当てはまる」という具体化

**設計のポイント**：
- 保育士の現場から見た「同じ場面」を語る（エビデンスとしての体験）
- 子どもの声・表情・動作まで書く（絵として浮かぶ描写）
- 「〇〇という子がいて——」という具体的な一人の子の話にする

**体験談の感情実況ルール**：
情報だけを語ると「読み物」になる。感情を実況すると「体験」になる。

```
情報だけ：「2歳の子どもは語彙が少ないため感情を行動で表現します」
感情実況：「2歳のRくんが噛んだ瞬間、相手の子が泣いた。Rくんは何が起きたかわからない顔をしてた。
           私はそこで気づいた。この子は『どうしていいかわからない』んじゃなくて、
           『言葉を持っていない』んだって」
```

---

### 第3章・有料核心：今日からの一手（有料）

**狙う感情**：「これをやれば私も変われる」という希望 + 具体的行動への衝動

**設計のポイント**：
- 抽象的なアドバイスは一切書かない
- 「明日の朝〇〇が〇〇したとき、こう言う」まで場面を特定する
- 「うまくいかなくても当然。でも3回やると変わり始める」で行動のハードルを下げる

**行動変容が起きる4条件**：
1. **場面特定性**：「いつ・どんな状況で使うか」が具体的（例：「夜泣きで起きた2時、抱き上げる前に——」）
2. **感情リハーサル**：「きっとこう感じると思いますが、そのときは——」（読者が事前に感情を体験できる）
3. **最小実験**：「まず1回だけ試してみて」（完璧にやらなくていいと伝える）
4. **結果の予告**：「劇的には変わらない。でも子どもの顔が少し変わる」（過大な期待を持たせない）

---

### おわりに（有料）

**狙う感情**：信頼感の完成 + 「この人の次の記事も読みたい」

**設計のポイント**：
- 「役立ちましたか」のような読者への問いかけは禁止
- ちゃむ。が「なぜこれを書いたか」の本音を1行で
- 読者への「あなたはもう十分やれてる」という肯定で終わる

## 3. 無料/有料の価値設計（具体的な判断フレーム）

20_note-kouzou-bunseki.mdの境界線設計を感情設計の視点で補足する。

### 無料に置くべきもの：「共感と認識」

読者が「この人は私のことをわかってる」と判断するための情報。
- 悩みの言語化（読者の体験を代弁する文章）
- 「なぜそれが起きるか」の発達段階説明
- ちゃむ。が同じ状況で感じた感情の実況

**無料の終わり方のルール**：
必ず「ここから先に何があるか」を具体的に示してから有料に移る。
```
❌ 「続きは有料記事で」
✅ 「ここから先では、3つのことをお伝えします。
    ①〇〇という気持ちになったときに使える言葉
    ②保育士として現場で実際に使っていたセリフ（verbatim）
    ③朝・夜・機嫌の悪い場面別の対応の分岐」
```

### 有料に置くべきもの：「解決と転換」

読者の「私にはできない」という感覚を「できるかもしれない」に変える情報。
- 今日から使える具体的なセリフ・スクリプト
- 「失敗した場合の次の手」（失敗を折り込んだ設計）
- ちゃむ。の体験の中で「転換点になった瞬間」の詳細
- 「保育士として現場で見てきた結果の数字・事例」

**有料の価値は知識量ではなく「私でもできる」感の量で決まる。**

## 5. Fable（感情設計AI）の使い方

### Fableに依頼する作業
- 感情動線の設計（各章の感情ターゲットを決める）
- はじめに の「共感の入口」の執筆
- 体験談の感情実況（「何を感じたか」の言語化）
- 有料核心の「転換点」の設計と執筆
- 行動変容パートの「感情リハーサル」の文章

### Fableへの依頼プロンプト（テンプレート）

```
以下のnoteテーマについて、感情設計を行ってください。

テーマ：[テーマ]
読者の悩み：[悩みの具体的状況]
ちゃむ。の実体験（あれば）：[体験]

お願いしたいこと：
1. 読者の旅路（3フェーズ）の感情ターゲットを設計してください
2. 「はじめに」の共感の入口を300文字で書いてください
3. 有料核心の「転換点」の場面を設計してください
   （「明日の朝〇〇したとき」レベルまで具体化してください）

制約：
- ちゃむ。は元保育士16年・3児の母・30代
- 子どもは現在全員小学生（育児体験は過去形）
- 押しつけない・「知識より共感」のトーン
- 行動変容4条件（場面特定性・感情リハーサル・最小実験・結果の予告）を満たすこと
```

### Sonnetが扱う作業（Fableに依頼しない）
- 型判定・価格決定・タイトル案生成
- 章立て・構成の設計
- NGチェック・カルテ管理・ステータス更新
- 発達段階の知識説明（情報パート）

## 6. 感情設計パターン3種（テーマに応じて選ぶ）

### パターンA：免責先行型（推奨・デフォルト）

```
流れ：「あなたのせいじゃない」→ 「なぜそうなるか」→ 「今日からの一手」
狙い：防御反応を解除してから情報を届ける
向く場面：自己嫌悪・罪悪感が背景にある悩み（怒鳴ってしまった・うまくいかない）
```

### パターンB：失敗共感→転換型

```
流れ：「私も同じで失敗した」→ 「でもこれを知って変わった」→ 「あなたも変われる」
狙い：ちゃむ。の実体験を前面に出して共感→信頼→希望
向く場面：ちゃむ。自身の体験が豊富なテーマ（寝かしつけ・イヤイヤ期・保育園トラブル）
```

### パターンC：代弁→深掘り→解放型

```
流れ：「こういうの、しんどいよね」→ 「なぜしんどいのかの構造を説明」→ 「それでいい・変えていい」
狙い：「しんどさの言語化」そのものに価値を置く
向く場面：「解決策より共感が欲しい」タイプの悩み（孤育て・夫との温度差・完璧主義）
```

## 7. 有料noteの価値を感じさせる文章の仕掛け

### 「読む前」の期待設計（タイトル・はじめに）
- タイトルに「読んだ後の自分の状態」を入れる
  （例：「読み終わったとき、今日の寝かしつけが少し違う気がしてくる記事」）
- はじめにの末尾に「この記事を読んだ後、あなたに残るもの」を3行で予告する

### 「読んでいる中」の期待維持
- 各章の最後に「次の章では〇〇します」を入れる（離脱を防ぐ）
- 「これを知ったとき、私は〇〇と思いました」という内側の反応を随時入れる

### 「読んだ後」の余韻設計
- おわりには「行動の最小単位」を1つだけ残す（「今夜寝かしつける前に、一度だけ試してみてください」）
- 感謝ではなく「あなたのことを信じている」で終わる
