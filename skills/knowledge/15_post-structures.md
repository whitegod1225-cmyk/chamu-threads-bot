# 15_post-structures.md ── 投稿構造テンプレート集

参考投稿から分解・抽象化した「投稿全体の型」。
フック・本文・CTA の組み合わせパターンを記録する。
12_hook-patterns.md（フック単体）・13_cta-patterns.md（CTA単体）と併用する。

> **使い方**：型を選び → ちゃむ。の文体（友人への語りかけ・元保育士視点）に置き換えて使う。
> 本文構造はそのまま借りて、語尾・人称・固有名詞はちゃむ。流に書き換える。

## 型S-1：時事脅威フック＋専門家権威＋リスト内URL型

**元ネタ**：ikuji_katarogu の手足口病投稿

### 構造

```
[フック] 〇〇が保育園で流行り始めたって…
[データ] ○月が感染ピーク／○歳以下が半数（厚労省 or 公的機関を引用）
[権威付け] 〇〇の専門家（母・友人・現場経験）に聞いたから絶対見てってほしい！
---（スライド2）---
[本文] 対策リスト ①〜⑤
　※リストの中に「〇〇するなら → [URL]」を自然に埋め込む
[締め] これ守るだけで〜。現場では〜って教えてくれた。（実績で約束）
```

### ポイント

- フックは「流行り始めた」「広まってる」など**進行形の脅威**で緊張感を出す
- 数字＋公的機関名で信頼性を付与（厚労省・小児科学会など）
- 権威者は「肩書き＋関係性」で語る（「元保育士の母」「現役の先生」）
- URLは独立行ではなく**リスト内に埋め込む**（対策の一部として商品を提示する）
- 締めは「これだけで〜できる」＋「実際に効いている実例」の2段階

### ちゃむ。への適用

- 権威：元保育士16年・現役児童支援員・3児の母 → 自分自身が権威になれる
- 脅威フック：感染症・発達・季節の体調変化 と相性◎
- リスト内URL：「〇〇するなら → コメントにリンク貼るね」でも代替可

## 型S-2：日常シーン報告＋懐疑→転換＋第三者引用型

**元ネタ**：na_mama7070ikuji のベビーチェア投稿

### 構造

```
[フック] 昨日、〇〇の家に行ったら（日常の第三者宅報告）
[商品登場] 〇〇を使ってて。
[懐疑] わざわざこれにする必要ある？って最初は心の中で思ってたんだけど…
---（スライド2）---
[転換] でも「〇〇だから〜」って言ってて（第三者の言葉を引用）
[読者の心が動いた理由を1行] 〜するだけで〜できる、という具体的使用感
[CTA] 欲しくて帰って速攻調べたわ / 気になって買ってみた（行動ベース）
[URL]
```

### ポイント

- フックは「昨日〜した」「先日〜に行ったら」など**過去の具体的な場面**で始める
- 「最初は懐疑的だった」を明示することで読者の反論を先取りし共感を作る
- **第三者の言葉を直接引用**することで発信者の押しつけ感を消す
- CTAは「私がこう感じた・こう動いた」という**感情ベースの行動報告**にする
  （「欲しくて速攻調べた」「気になって即注文した」）
- 商品の長所は「第三者が使っている場面」に乗せて伝える

### ちゃむ。への適用

- 第三者：「ママ友」「保育士仲間」「保育園の先生」「支援員の先輩」
- 転換：保育士時代の経験や現場知識が「懐疑→納得」の説得力になる
- CTA例：「帰ってから気になって調べたら」「使いやすそうで速攻チェックした」

## 型S-4：速報興奮型

**元ネタ**：non_hokuo のイッタラアウトレット投稿

### 構造

```
[フック] え、待って待ってーーッッ！（興奮・驚き）
[発見報告] 〇〇に〇〇が追加されてる！
---（スライド2）---
[希少性の示唆] これだけの理由で〇〇？というのもたくさんあってびっくり
[緊急CTA] 争奪戦の予感…早めにチェックしてね
[URL]
```

### ポイント

- フックの感嘆符・繰り返し（「待って待って」「ーーッッ」）で発信者の興奮を可視化する
- 「自分が今まさに発見した」リアルタイム感が読者を引き込む
- 在庫・期間の希少性を明示しつつ、「争奪戦の予感」など軽い断言で緊張感を出す
- 商品の長所を詳しく説明しない（速報として機能する量に留める）

### ちゃむ。への適用

- セール開始・再入荷・期間限定商品の瞬間を捉えて使う
- 「待って待って」「え、これすごくない？」くらいのトーンがちゃむ。に合う
- 感嘆符の連打はちゃむ。トーンでは控えめに（1〜2個まで）

## 型S-5：逆説後悔型

**元ネタ**：rakuko_jitan_40s のバランスドーム投稿

### 構造

```
[フック] 本当に後悔してます……（ネガティブに見える出だし）
[逆説展開] 〇〇が〇〇になって・〇〇になって（良い変化の連鎖報告）
---（スライド2）---
[追加の良い効果] さらに〇〇という効果も（予想外の副次効果）
[タイミングCTA] 今年の〇〇に活躍してもらいます！（季節・時期への紐付け）
[URL]
```

### ポイント

- 「後悔してます」「嫌になった」など一見ネガティブなフックで読者を引き込み、スライド2で「実は良い意味」と転換する
- 効果の連鎖（体力→体幹→姿勢→イライラ切り替え）を「そして…」「さらに…」で積み重ねて読者を引き込む
- CTAは商品説明ではなく「〇〇の季節に使います」という自然な生活への組み込みで締める

### ちゃむ。への適用

- フック例：「本当にやられた…」「やめとけばよかった…（いい意味で）」
- 効果の連鎖は保育士視点で「発達・体幹・感情調整」のキーワードと組み合わせると強い
- 梅雨・夏休み・年度替わりなど季節タイミングのCTAと相性◎

## 型S-6：ターゲット断言まとめ型

**元ネタ**：coco.mama.ikuji の夏の赤ちゃん対策投稿

### 構造

```
[フック] 〇〇のママへ（ターゲットを断言して呼びかける）
①〜⑩ 番号付きリストで有益情報を提供
　各項目の中に「〇〇なら『商品名』」の形で商品を自然に埋め込む
---（スライド2〜）---
各項目を掘り下げ（詳細・使い方・注意点）
[URL]
```

### ポイント

- 「〇〇のママへ」でターゲットを明確に呼び込む（読者が「自分のことだ」と感じる）
- 番号付きリストは10項目前後で「これだけ知っておけば大丈夫」という網羅感を出す
- 商品名は「❛商品名❜」のように引用符で括って情報の一部として提示する（広告感を薄める）
- 有益情報が主役で商品は脇役という構造にする

### ちゃむ。への適用

- フック例：「初めて夏を迎えるママへ」「育休明けを控えたママへ」「1歳前後のママへ」
- 保育士16年の知識でリストを構成することで信頼性が格段に上がる
- 項目数は5〜8が読みやすい（10は多い）

## 型S-7：危機感→比較解決型

**元ネタ**：coco_chan_babyitem の骨盤ベルト投稿

### 構造

```
[フック] 〇〇を放置すると〇〇になるんです……！（危機感・損失回避）
[専門家引用] 先生に「〇〇しなさい！」と言われた（権威による後押し）
[読者の疑問を言語化] でも「AとB、どっちがいいの？」って迷うよね
---（スライド2〜）---
[比較解説] A（履くタイプ）のメリット → [URL]
[比較解説] B（巻くタイプ）のメリット → [URL]
```

### ポイント

- 放置した場合の最悪シナリオを具体的に書くことで「今すぐ対処しなければ」感を作る
- 専門家（先生・医師・保育士）の言葉で権威付けしてから商品に進む
- 「どっちがいいの？」という読者の迷いを先取りして言語化することで信頼感が増す
- A vs B 比較型はスライドを分けることで各タイプにURLを自然に配置できる

### ちゃむ。への適用

- 危機感フックは煽りすぎず「〜になることがあるんだよね」程度に和らげる
- 専門家引用：「保育士時代の先輩に言われた」「小児科の先生が言ってた」で使える
- 比較型：「〇〇派 vs △△派」「0歳向け vs 1歳〜」など月齢・用途で分けると有効

## 型S-8：衝動購入報告型

**元ネタ**：sakura.mom_ikuji のアンパンマン絵本投稿

### 構造

```
[フック] 見つけてすぐ買った（衝動購入の事実から始める）
[買った理由] この値段で〇〇は反則。絶対ハマるやつ。
---（スライド2）---
[内容紹介] 〇〇たっぷり・〇〇たっぷり（畳み掛け）
[用途提案] 〇〇にも◎・〇〇にも◎
[URL]
```

### ポイント

- 「買った」という完了した行動から始めることで読者に「なぜ？」を自然に作る
- 「反則」「やばい」など価格とクオリティのギャップを強調する言葉を1語入れる
- 「〇〇たっぷり」の畳み掛けは2〜3回で十分（リズムで読者を引き込む）
- 用途提案は「雨の日・外出・プレゼント」など具体的なシーンを2つ以上提示する

### ちゃむ。への適用

- フック例：「見つけて秒で買ったやつ」「気づいたらカートに入れてた」
- 価格感の表現：「この値段でこれは」「コスパどうなってるの」くらいが自然
- ちゃむ。トーンでは「反則」「やばい」は1投稿に1回まで（乱用するとAI感が出る）

## 型S-9：笑えるエピソード導入型

**元ネタ**：ao_kurashi100 のウサギライト投稿

### 構造

```
[フック] 〇〇が「〇〇」だから、冗談で〇〇を提案したんだけど（日常の笑えるエピソード）
[家族・友人の予想外の反応] 真剣な顔で「〇〇？〇〇？」って（真剣に受け取られた）
---（スライド2）---
[笑いを引きずりながら商品説明] これはたぶん〇〇にいる（予想）
[商品の機能を自然に説明] 光るし・吊るせるし・耳まで曲がるの普通に可愛い
[URL]
```

### ポイント

- 笑えるエピソードが「つかみ」になり、商品説明に入る前に読者の心が柔らかくなる
- 家族・友人の「リアルな反応」を引用することで発信者の主観を薄める
- 「これはたぶん〇〇にいる」のような軽い予言で読者に続きを読ませる
- 商品スペック（光る・吊るせる・曲がる）を会話の流れの中で自然に並べる

### ちゃむ。への適用

- 「子どもに冗談で言ったら本気にされた」「夫に相談したら意外な反応が返ってきた」
- 笑いのトーンはちゃむ。に合う（保育士時代の子どものエピソードとも組み合わせやすい）
- 商品説明はエピソードのテンションを維持しながら自然に接続する

### バリエーション：家族すれ違いオチ型

**元ネタ**：moku_kurashi100 の「コロッケ見に行ったら…ごめん、お母さん」

「〇〇って言うから見に行ったら、ごめん…」という家族のすれ違い話でオチを作り、その先に商品がある。

```
[フック] 〇〇が「〇〇だから見て」って何回も言うから見に行ったら、
[落ち] ごめん…〇〇（誤解していたことへの謝罪的笑い）
---（スライド2）---
[実際の話] 〇〇じゃなくて、〇〇を見せたかったらしい
[共感の締め] たしかに〇〇。見せたくなる〇〇の気持ちもわかる（笑）
[URL]
```

> 「ごめん、お母さん」のように発話ターゲットを明示することで場面が鮮明になる。
> オチが笑えるほど商品説明への自然な橋渡しになる。
> ちゃむ。例：「子どもが『これ見て見て』ってうるさかったから見たら、ごめん〇〇ちゃん…」

## 型の選び方

### A. 状況から選ぶ

| 状況 | 使う型 |
|------|--------|
| 季節の感染症・育児ニュースに乗る | S-1（時事脅威＋専門家権威） |
| 自分が実際に試した or 見た商品を紹介 | S-2（日常シーン＋懐疑→転換） |
| 専門知識で有益情報を提供する | S-3（有益情報ハック） |
| 再入荷・セール開始・限定品を速報で出す | S-4（速報興奮型） |
| 使い続けた効果の連鎖を伝える | S-5（逆説後悔型） |
| 月齢・時期に合わせた網羅情報を届ける | S-6（ターゲット断言まとめ型） |
| AとBを比べて選んでもらう | S-7（危機感→比較解決型） |
| 見つけた瞬間の衝動感をそのまま出す | S-8（衝動購入報告型） |
| 笑えるエピソードを入り口にする | S-9（笑えるエピソード導入型） |
| 自分の失敗談から商品に導く | S-2変形（懐疑を「失敗」に置き換え） |
| 商品・ブランドへの問いかけで称賛する | S-15（問いかけ→天才称賛型） |
| フックで誤解させてスライド2でオチを出す | S-13（ミスリード→ネタバラシ型） |
| 自分の具体的な悩みを直接開示して解決商品に導く | S-14（自己悩み開示→解決報告型） |
| 外出先で見かけた商品への衝動感を出す | S-10（目撃→二度見→謝罪転換型） |
| コラボ・限定品を「困る」トーンで紹介する | S-11（ブランド擬人化ツッコミ型） |
| 話題商品と比べて自分の本命を宣言する | S-12（話題商品比較→優勝宣言型） |
| 目撃→会話→第三者の想いで感情を落とす | S-16（目撃→会話→感情落ち型） |
| 値上げ・廃番ニュースを共感の入り口にして「今が狙い目」に転換する | S-19（ネガティブニュース→緊急性転換型） |
| 新発見した商品への「出会うのが遅かった」後悔を伝える | S-17（感情爆発後悔型） |
| セール・まとめ買い特典を友人感覚で共有する | S-18（読者共犯型） |
| 本音を言いながらフォロワーと信頼を積む | E-1（炎上覚悟共感型） |
| 怒り・疲れ・無気力の本質を専門家として語る | E-2（専門家問題再定義型） |
| 話題のメソッドを科学的に解説して権威化する | E-3（著名人文脈乗り×科学的裏付け型） |
| バズ直後に共感を延長してコミュニティ感を作る | E-4（バズ報告エンゲージメント維持型） |

### B. 商品特性から選ぶ（URLをもらったらここで型を決める）

URLを受け取ったとき、まず商品の「一番の強み」を1つ決めてから対応する型を選ぶ。

| 商品の一番の強み | 第1候補 | 第2候補 | 判断のヒント |
|---------------|--------|--------|------------|
| **見た目・デザインが可愛い** | S-17（感情爆発後悔型） | S-15（天才称賛型） | 「なぜ早く見つけなかった」「誰が作ったの」が出るなら |
| **機能・問題解決（育児の悩みを解消する）** | S-14（悩み開示→解決報告型） | S-7（比較解決型） | 「〇〇で悩んでたけど」から始められるなら |
| **コスパ・価格の安さが強み** | S-8（衝動購入報告型） | S-18（読者共犯型） | 「この値段でこれは」という驚きがあるなら |
| **定番・有名商品の代替品** | S-12謝罪乗り換え型 | S-12危なかった発見型 | ユニクロ・IKEA・西松屋など定番に対比できるなら |
| **知る人ぞ知る隠れ名品** | S-2（日常シーン→転換型） | S-3（有益情報ハック型） | 「保育士仲間に教えてもらった」から始められるなら |
| **セール中・限定・再入荷** | S-4（速報興奮型） | S-18（読者共犯型） | 今この瞬間の価格優位が主な理由なら |
| **値上げ・廃番が決まっている** | S-19（ネガティブニュース→緊急性型） | S-4（速報興奮型） | 期限・締め切りを共感で伝えられるなら |
| **長期使用・リピート実績がある** | S-5（逆説後悔型） | S-2（日常シーン型） | 「使い続けてわかったこと」が語れるなら |
| **子どもの反応・エピソードがある** | S-9（笑えるエピソード導入型） | S-13（ミスリード→ネタバラシ型） | 笑えるか驚ける子どものエピソードがあるなら |
| **ブランド自体を推したい（複数商品）** | S-11（ブランド擬人化ツッコミ型） | S-15まとめ版 | 同ブランドの商品を3つ以上紹介できるなら |

> **迷ったときのデフォルト**：S-14（悩み開示→解決報告型）。育児商品全般で機能しやすく、ちゃむ。の保育士視点と最も組み合わせやすい。

## 型S-10：目撃→二度見→謝罪転換型

**元ネタ**：iro_to_kurashi の「電車で前に座ってた人が〜」投稿

### 構造

```
[フック] これこれ！〇〇で前に〇〇が〇〇してて…（外出先での目撃報告）
[画像] 商品の写真（文字なし or 最小限）← スライド1〜2は画像が主役
---（スライド2）---
[謝罪転換CTA] ごめんなさい。可愛過ぎて二度見しました
[URL]
```

### ポイント

- S-2（日常シーン報告型）の外出版。「ママ友の家」→「電車・街中・店頭」に置き換えた型
- フックの「これこれ！」は読者に「何が？」を作る強い引き
- スライド1〜3は画像が主役。文章は最小限（商品の魅力を言葉で説明しない）
- 「ごめんなさい。可愛過ぎて〇〇しました」はユーモラスな謝罪形式で衝動感を表現する
- テキストが少ない分、画像のクオリティで勝負する投稿

### ちゃむ。への適用

## 型S-15：問いかけ→天才称賛型

## 型S-18：読者共犯型（ごにょごにょ）

- 「ごにょごにょ」はちゃむ。トーンに合う（友人感が出る）
- セール中の「まとめ買い特典」「クーポン重ねがけ」「ポイント倍デー」に使いやすい
- 楽天マラソン・スーパーSALE期間中の速報投稿に特に向く

### S-4（速報興奮型）との違い

| | S-4 | S-18 |
|--|-----|------|
| 感情 | 発見の興奮（待って待って！） | 信じられない驚き（は？） |
| 読者との関係 | 発信者→読者への情報共有 | 一緒に考える・共犯感 |
| 締め | 「早めにチェックして」 | 「ごにょごにょ」の言いかけ |

# エンゲージメント・信頼構築型（商品なし）

アフィリエイトリンクを含まず、フォロワーとの信頼・共感を積み上げる投稿の型。
バズ狙いよりも「この人をフォローし続けたい」と思わせることが目的。

## 型E-1：炎上覚悟共感型

**元ネタ**：ebi._.ikuji の「ママが満たされていることが大事だよ」投稿

### 構造

```
[フック] 炎上覚悟だけど、〇〇が大事だよ。（覚悟宣言で注意を引く）
[現状肯定リスト] 「〇〇しよう」「〇〇してあげよう」（読者がすでにやっていることを列挙）
---（スライド2）---
[反問の畳み掛け] 〇〇は誰がしてくれるの？ × 3回（読者自身の不在を気づかせる）
[比喩で概念化] 愛情ってコップに注がれた水みたいなもので…（視覚的な比喩）
[真実の提示] なのに〇〇を満たす方法はほとんど語られてない
[解決提案] 〇〇を作ることに気を向けてみて（具体的・小さいアクション）
```

### ポイント

- 「炎上覚悟だけど」は注意喚起であり、「でも言わなきゃいけない」という誠実さの表明
- スライド1で読者の「頑張り」を全力で認めてから転換する（批判ではなく寄り添い）
- 反問の畳み掛けは同じ構造を3回繰り返す（テンポが大事）
- 比喩（コップの水）は抽象的な感情を誰でも分かる言葉に落とし込む
- 解決提案は「誰かに頼れ」ではなく「自分でできる小さいこと」にする

### ちゃむ。への適用

- フック例：「炎上覚悟だけど、保育士から見て正直に言います」「言いにくいけど、現場では当たり前のことを言う」
- 反問例：「子どもの感情は誰が守るの？→ じゃあ、あなたの感情は？」
- 保育士16年の現場知識が「炎上覚悟」の重みと説得力になる

## 型E-2：専門家問題再定義型

**元ネタ**：ebi._.ikuji の「ママの怒りが抑えられないのはSOSサインです」投稿

### 構造

```
[フック] 危険です。〇〇は〇〇のSOSサインです。（断言と危機感）
[状況の描写] 〇〇、〇〇、〇〇…（読者の日常を列挙）
[真の原因の提示] 単に〇〇の話じゃなくて、【〇〇が問題】（一般論を否定して本質を出す）
[科学・専門知識の裏付け] 人は〇〇がないと〇〇する力が落ちる
---（スライド2）---
[「必要なのはこれじゃない」の転換] 「〇〇を頑張ろう」という話ではなくて
[本当の解決] 「〇〇を減らす」を考えること
[具体的行動] たとえば、〇〇する・〇〇でいいことにする（小さい3つ）
[締め] そういった〇〇のサインを見逃さないでね
```

### ポイント

- 「危険です」「SOSサインです」は専門家だから言える断言フレーム
- 「これじゃない→本当にこれ」という二段構えの転換が信頼構築の核心
- 解決策は「頑張り方を変えるのではなく、頑張る量を減らす」のように既存の努力を否定しない
- 具体的行動は小さく・すぐできるものを3つ以内で出す（ハードル最低）

### ちゃむ。への適用

- フック例：「これ、ただの甘えじゃないよ。心のSOSサインだよ」「保育士から見て、これは危険信号です」
- 専門知識の出し方：「16年現場にいてわかったんだけど、〇〇の問題は実は〇〇にある」
- 転換フレーム：「必要なのは〇〇じゃなくて、〇〇を見直すこと」

## 型E-3：著名人文脈乗り×科学的裏付け型

**元ネタ**：ebi._.ikuji の「コール&レスポンス育児」投稿

### 構造

```
[フック] 〇〇や〇〇が使ってるって話してた「〇〇」が楽しすぎる（著名人の話題に乗る）
[具体例] こんな感じで → [会話形式で実演]
[科学的裏付け] 実は〇〇的にめちゃくちゃ理にかなってて
---（スライド2）---
[仕組みの説明] 脳内では❶〜❸（番号付きで視覚的に）
[専門用語を平易に] 〇〇って、〇〇のことで（定義を一言で添える）
[既存方法との比較優位] 「〇〇！」と叱られるより〜（従来との違い）
```

### ポイント

- 著名人（ミキティ・横澤夏子など）の名前を出すことで「話題の情報」として権威付け
- 会話形式（ママ：「〇〇？」→ 子ども：「〇〇！」）で読者がすぐ試せるイメージを作る
- 科学用語（ワーキングメモリ・前頭前野）を出した直後に平易な説明を添える
- 「叱るより〇〇の方が良い」という比較で既存の苦労を肯定しながら転換する

### ちゃむ。への適用

- 著名人の代わりに「フォロワーさんから教えてもらった」「支援センターで先生が話してた」でも機能する
- 発達・感情調整・ワーキングメモリなど保育士知識と相性◎
- 「これって実は〇〇的に理にかなってる」フレームはちゃむ。の発信に自然に使える

## 型E-4：バズ報告エンゲージメント維持型

**元ネタ**：ebi._.ikuji の「え、まってまって、一個前の投稿の反応が凄いね！」投稿

### 構造

```
[驚き報告] え、まってまって、前の投稿の反応が凄いね！
[読者の状態への共感] みんな〇〇なのね。毎日本当に頑張ってるね、お疲れさま。
[ねぎらい] ありがとう🫶
[小さい提案] 〇〇したら、自分のための時間も作ろうね
```

### ポイント

- 単体投稿ではなく「前の投稿がバズったとき」専用の型
- 数字・いいね数ではなく「反応」という言葉でコミュニティ感を演出する
- 読者の行動（寝かしつけ・家事）を想像して声をかけることで親密感を作る
- 投稿の長さは短くていい（2〜3行）。余韻を壊さず共感を延長する役割

### ちゃむ。への適用

- 「前の投稿、思ったより反応もらえて嬉しかった」
- 「こんなに共感してくれる人がいたんだって気づいた」
- バズ直後に必ず出すことで「読んでくれてありがとう」という誠実さが伝わる

### バリエーション：イベント問いかけエンゲージメント型

**元ネタ**：muji_ota の「楽天スーパーセール、何買う？」

セール・イベント期間中に読者に問いかけてコメントを引き出す。商品紹介なし。

```
[問いかけ] 〇〇（イベント名）、何買う？
[自分の本音でオチ] 〇〇ありすぎて、ま、いっか。ってなりそう。
```

> セール直前に出すと「何が良い？」コメントが集まり次の投稿ネタになる。
> 2〜3行で完結させる。長く書かない。
> ちゃむ。例：「楽天セール、何買うか決めた？わたしは悩みすぎて全部カートに入れてる件」

## 訴求の本質：「未来を売る」4ステップ（モジル式）

> 出典：モジル note「アフィリエイトを超簡単に攻略しちゃう方法」
> **人は商品の中身（機能・スペック）では動かない。「これを使ったら自分は変われる」と未来がはっきり見えた時に財布を開く。**

### 売ってはいけないもの vs 売るべきもの

| ❌ 売ってはいけない | ✅ 売るべき |
|-----------------|----------|
| 成分・機能・スペック | 使った後の未来・ゴール |
| 「〇〇が入ってます」 | 「〇〇が手に入る」 |
| 商品の仕様説明 | 読者の不満が消える場面 |

> おにぎりの例：「新潟産の米で十勝の鮭使用」← お腹が空いてる人には不要。
> 「噛んだ瞬間に塩分と香りが口を満たす」← これが"未来の体験"を売っている。

---

### 4ステップ：未来を売る本文構成

```
（1）読者の〈今〉を言葉にする
　　→ 今の不満・コンプレックス・諦め・繰り返してる失敗

（2）〈理想の未来〉をありありと描く
　　→ 「こうなれたら最高」という姿。具体的な場面・感情で

（3）〈差分〉を感じさせる
　　→ 「今のまま」と「こうなれる」のギャップを読者に自覚させる

（4）その橋渡しが商品だと伝える
　　→ 「このギャップを埋めるのがコレ」
```

**商品は〈主役〉ではなく、〈過去から未来へ渡る橋〉。**
この順番に変えるだけで、同じ商品でも成約率がガラッと変わる。

---

### ちゃむ。への適用例

| 商品 | ❌ 機能説明 | ✅ 未来訴求 |
|-----|-----------|----------|
| 防臭袋 | 3層フィルムで臭いを遮断 | 玄関に出してもにおわない。ゴミの日まで気にならない |
| バランスドーム | 体幹トレーニングができる | 外遊びできない日でも、子どもが自分で発散してくれる |
| 知育おもちゃ | 木製・知育効果あり | 「ねえ見て見て」ってくるから、なんか一緒にいる時間が増えた |

---

### 組み合わせるべき型

- S-5（逆説後悔型）：変化の連鎖が「未来の描写」になる
- S-14（自己悩み開示→解決報告型）：ステップ(1)→(4)をそのまま投稿構造にできる
- E-1（炎上覚悟共感型）：読者の〈今〉の肯定→本当に必要なものへの転換

## 構造横断メモ

- **スライド分割の原則**：フック＋文脈をスライド1、商品情報・本体・URLをスライド2以降に置く。スライド1で「続きが気になる」状態を作ってから読ませる
- **URL配置**：コメント欄 or リスト内埋め込み。独立した「↓こちら↓」行はちゃむ。トーンでは広告感が出やすい
- **締め方の種類**：①実績で約束（「ほぼかからない」）②感情行動報告（「速攻調べた」）③季節タイミング（「今年の梅雨に活躍してもらいます」）④逆転発想（「いっそ主役にするのが正解かも」）
- **スライド数の目安**：2枚が基本。S-6・S-7など情報量が多い場合は3〜5枚まで。1枚に詰め込まない

## 投稿制作の3原則（sukimarisu式）

> 出典：sukimarisu2415（Threads）

---

### 原則1：数字を1つ入れる（「ふーん」→「え、それほしい」）

抽象的な言葉を具体的な数字に置き換えるだけで、読者の反応が変わる。

| ❌ 抽象 | ✅ 数字あり |
|--------|----------|
| 時短グッズおすすめ | 朝の準備が15分→8分になったグッズ |
| 寝かしつけにいいよ | 寝かしつけ45分が10分になった |
| 離乳食ラクになった | 日曜30分仕込むだけで平日全部レンチン |

> ちゃむ。への適用：保育士16年・3人子育ての経験は「自分の数字」に変換できる。
> 「子ども3人試した」「6年使い続けてる」「夜泣きが3日で落ち着いた」など体験値を数字で出す。

---

### 原則2：ピークエンドの法則（1行目と最後だけ本気で書く）

人の記憶に残るのは「いちばん盛り上がった瞬間」と「最後の印象」だけ。途中は7割でいい。

```
1行目 → 「あるある」か「えっ？」で止める（保存・続きを読む動機を作る）
最後  → 「同じ人いる？」など未完了感を残す（コメントが来る・記憶に残る）
真ん中 → 7割の完成度で出す。完璧にしなくていい
```

> ちゃむ。への適用：
> - フックはすでに型で押さえてある（12_hook-patterns.md）
> - 締めの一文に「同じ人いる？」「これあるある？」を意識的に入れる
> - 真ん中に時間をかけすぎない。フック→要点→締めの順で構成する

---

### 原則3：ネタは「昨日何があった？」で考える

「何を書こう」という発想を捨てる。育児してる時点でネタは毎日自動生成される。

```
離乳食こぼされた → ネタになる
寝かしつけ45分かかった → ネタになる
子どもが靴を左右逆に履いてた → ネタになる
```

「書くことがない」のではなく「書いていいと思っていない」だけ。

> ちゃむ。への適用：
> - 「昨日の育児エピソード」を入り口にする型は S-9（笑えるエピソード）・S-2（日常シーン報告）がある
> - エピソードをメモしておき、型に当てはめるだけで投稿になる
> - 「3児の母×元保育士」の日常は読者にとってコンテンツそのもの
