# 12_hook-patterns.md ── フック集（1行目パターン）

> **このファイルの使い方**：投稿の1行目を書くとき、まず「通常投稿フック」から型を選ぶ（投稿の8割はこっち）。
> アフィリ投稿のときだけ「アフィリ向けフック」を見る。
> どの型でも、ちゃむ。の口語（タメ口・文末「。」なし）に必ず言い換えてから使う。

229本の実績分析（2026年6月時点）に基づく改訂版。
伸びた投稿の共通点：**具体的な場面が浮かぶ・今日から使える・罪悪感が軽くなる**。

# 第1部：通常投稿フック（メイン・投稿の8割）

## 1.「〇〇の代わりに言う言葉」型 ★最強（6月最高1,850views）

「今日から使える一言」がもらえると読者が確信できる型。
親が毎日言ってしまう言葉（早くして・ダメ・泣かないで）を入り口にする。

| 書き出しテンプレ | 例文 |
|---------------|------|
| 「〇〇」の代わりに言ってた言葉 | 「早くして！」の代わりに保育士が言ってた言葉がこれ |
| 「〇〇」って言いそうになったら | 「ダメ！」って言いそうになったら、この一言に変えてみて |
| 保育士は「〇〇」って言わない | 保育士は「泣かないで」って言わない。代わりに言うのは |
| 私が「〇〇」をやめて変わったこと | 「ちゃんとして」をやめたら、朝の支度が10分早くなった |
| 〇〇な時、現場で使ってた言葉 | 癇癪が始まった時、現場で毎日使ってた言葉があるんだけど |

**使い方メモ**
- 対象の言葉は「読者が今日すでに言ってしまった言葉」を選ぶと刺さる
- 代わりの言葉は本文の早い段階で出す。出し惜しみすると離脱される
- 「〜7選」など数字と組み合わせると保存が伸びる（例：「癇癪を止める言葉」7選）

## 2. 概念再定義型 ★罪悪感を溶かす（1,446views）

「実はそれ〇〇じゃなくて〇〇なんだよ」で、親の罪悪感やイライラの見え方を変える型。
子どもの行動の再定義と、母親自身の行動の再定義、両方いける。

| 書き出しテンプレ | 例文 |
|---------------|------|
| 〇〇は「〇〇」じゃなくて「〇〇」 | 2歳のイヤイヤは「わがまま」じゃなくて「脳の工事中」って知ってた？ |
| 〇〇って実は〇〇なんだよね | 「ママ見て見て」って実は、愛情の確認作業なんだよね |
| それ、〇〇じゃないよ | 子どもの前で泣いちゃうの、母親失格じゃないよ |
| 〇〇は手抜きじゃなくて〇〇 | 冷凍うどんの夜は手抜きじゃなくて「明日も母をやるための作戦」だから |
| 〇〇に見えるけど、あれ〇〇だから | 兄弟ゲンカ、仲が悪いように見えるけど、あれ社会性の練習だから |

**使い方メモ**
- 「悪いこと」→「意味のあること」への変換が基本。逆方向（良いこと→実は悪い）は不安を煽るので使わない
- 再定義のネーミングは体で覚えてる言葉にする（脳の工事中・充電切れ・作戦）
- 語尾「って知ってた？」は問いかけになってコメントも拾いやすい

## 3. 告白・報告型 ★権威崩し（1,040views）

「専門家なのにできてなかった」「本音を言うと」で距離を縮める型。
16年保育士×3児の母のギャップがそのままフックになる。

| 書き出しテンプレ | 例文 |
|---------------|------|
| 〇年〇〇なのに、〇〇だった | 16年保育士やってたのに、自分の子どもには怒鳴ってた |
| 「〇〇」って1日〇回言ってた私が | 「早くして！」って1日20回言ってた私が、言わなくなった方法 |
| ぶっちゃけると、 | ぶっちゃけると、保育士も家では完璧じゃない |
| 黙ってたけど、 | 黙ってたけど、私も子どもの前で泣いたことある |
| 正直に言うね | 正直に言うね。「叱らない育児」、現場では無理だった |

**使い方メモ**
- 告白の中身は「読者もやってること」にする。読者が経験してない失敗の告白は他人事になる
- 数字を入れると告白がリアルになる（1日20回・16年・3人全員）
- 告白で終わらず「でも変われた」「だから大丈夫」まで書く。落として終わりはNG

## 4. カオスな日常あるある型（647views）

視覚的に浮かぶ具体場面で「うちだけじゃなかった」を作る型。
抽象的な「育児大変だよね」ではなく、映像が浮かぶ一場面を切り取る。

| 書き出しテンプレ | 例文 |
|---------------|------|
| 〇〇しながら〇〇して〇〇する朝 | 靴下片方探しながら連絡帳書いて「早く食べて」って言う朝 |
| 〇時、〇〇してたら〇〇 | 朝7時、味噌汁よそってたら「今日図工で牛乳パックいる」って言われた |
| うちだけ？〇〇するの | うちだけ？お風呂上がりの子ども捕まえるのに全力疾走するの |
| 〇〇した5秒後に〇〇 | 「もう知らない！」って言った5秒後に「ママ大好き」って言われた |
| 3人いると〇〇が〇〇になる | 3人いると「静かだな」が一番こわい時間になる |

**使い方メモ**
- 時刻・場所・セリフのどれか2つを入れると映像が浮かぶ
- オチや教訓は無理につけなくていい。「わかる」だけで成立する型
- 「うちだけ？」はコメント誘導も兼ねる万能語尾

## 5. 場面描写×救い型

読者の「一番しんどい瞬間」をピンポイントで描写して、そのまま許可を出す型。
再定義型の姉妹パターン。夜投稿と相性がいい。

| 書き出しテンプレ | 例文 |
|---------------|------|
| 〇時、〇〇しながら〇〇するの、もうやめていいよ | 夜9時、寝顔見ながら「今日も怒りすぎた」って泣くのもうやめていいよ |
| 〇〇してる時点で、〇〇だよ | 「怒りすぎたかな」って考えてる時点で、ちゃんといいお母さんだよ |
| 〇〇な日があってもいい | 一日中イライラして終わる日があってもいい |
| 今日〇〇できなかった人へ | 今日、絵本読んであげられなかった人へ |

**使い方メモ**
- 場面は「夜・寝顔・ひとり時間」が鉄板。読者がスマホを見てる状況と重なる
- 「〜べき」「〜しましょう」は絶対NG。許可・免罪のトーンで書く
- 救いの根拠に保育士の視点を1行添えると説得力が出る

## 6. 数字×専門性型

「現場で使ってた」「16年で分かった」に具体的な数字を掛け合わせる型。
保存されやすいまとめ投稿の1行目に向く。

| 書き出しテンプレ | 例文 |
|---------------|------|
| 保育士が現場で毎日使ってた〇〇、〇選 | 保育士が現場で毎日使ってた「癇癪を止める言葉」7選 |
| 16年で分かった、〇〇の子への声かけ | 16年で分かった、切り替えが苦手な子への声かけ |
| 〇歳の「〇〇」、実はこうだった | 4歳の「なんで？攻撃」、実は答えなくてよかった |
| 3人育てて結論出た。〇〇は〇〇 | 3人育てて結論出た。トイトレは焦った分だけ長引く |

**使い方メモ**
- 数字は「7選」「16年」「1日20回」など奇数・実数が強い。「3つの理由」はタイトル臭くなるのでNG
- リスト投稿でも1行目は「解説」じゃなく「現場で使ってた」など体験ベースにする
- ❶❷❸で仕組みを解説する構成は沈む。リストは「そのまま使える言葉・行動」だけにする

## 補助：つかみワード（どの型にも合わせられる書き出し）

| 書き出し | 機能 |
|---------|------|
| ぶっちゃけると、 | 本音トーン。告白型の入り口 |
| 黙ってたけど、 | 秘密の共有感 |
| これ言うと驚かれるんだけど | 意外性の予告。再定義型と好相性 |
| うちだけ？ | あるある型＋コメント誘導 |
| 正直に言うね | 覚悟の表明。専門家の本音に |

つかみワードは1投稿1個まで。毎回同じ書き出しにすると「型がバレて」効果が落ちるのでローテーションする。

# 第2部：アフィリ向けフック（投稿の2割）

通常投稿フックの応用でアフィリも書ける。以下は商品紹介に特化した型だけ残す。

## A. 日常シーン報告型

第三者が使ってる場面から入る。押しつけ感ゼロで商品を登場させられる。

| フック例文 | 使えるシーン |
|-----------|------------|
| 昨日ママ友の家に行ったら〇〇使っててさ | 訪問先での発見 |
| 支援センターで見かけたママが使ってて | 育児現場での目撃 |
| 保育園の先生が使ってるの見て気になった | プロの使用・信頼性 |

## B. 逆説・ミスリード型

ネガティブや誤解を装って引き込み、2行目でオチ。週1〜2本の笑いポスト向け。

| フック例文 | 使えるシーン |
|-----------|------------|
| 正直後悔してる。なんでもっと早く買わなかったんだろ | 購入が遅れた後悔 |
| やめとけばよかった…子どもが夢中になりすぎて | 成長・変化系 |
| 保育士が〇〇してるって言ったら引かれそうだけど | 意外に見える行動 |

## C. 体験×数字型

実体験の数字で信頼を作る。ちゃむ。は「3人で使い倒した」が言える。

| フック例文 | 使えるシーン |
|-----------|------------|
| 3人で使い倒して結論出た | 長期・複数人レビュー |
| 3時間が15分になったやつ | 時短商品 |
| 何回リピートしたか数えてない | 高リピート商品 |

## D. 比較優勝宣言型

話題の選択肢を認めてから本命を出す。旬の話題への乗り方として。

| フック例文 | 使えるシーン |
|-----------|------------|
| 〇〇も良いけど、うちでは〇〇が優勝だった | 人気商品との比較 |
| バズってるのわかるけど、正直こっちのほうが合ってた | SNS話題商品との比較 |

## E. 速報型（セール・再入荷のときだけ）

| フック例文 | 使えるシーン |
|-----------|------------|
| 気づいたら復活してた！これは早めに見て | 再入荷 |
| セール始まったの今知って急いでる | セール開始 |

「優勝」「反則」「やばい」は1投稿1回まで。感嘆符は2個まで。

## F. ターゲット指定系（oji_to_moji式・育児向けに変換）

対象読者を1行目で名指しする型。「自分のことだ」で手が止まる。

| フック例文 | 使えるシーン |
|-----------|------------|
| 〇歳のママへ | 月齢・年齢ピンポイント訴求 |
| 寝かしつけで毎晩30分以上かかってる人、見て | 悩み特定型 |
| 離乳食の進め方で迷ってるママに届けたい | 時期×悩み組み合わせ |
| 保育園入りたての子がいるお母さんへ | 入園・慣らし期 |

ちゃむ。での注意：「〇〇なの私だけ？」をターゲット指定と組み合わせると共感×指定の二重効果が出る。

## G. 感情系（ちゃむ。トーン版）

スクロールを止める短い感情表現。oji_to_mojiの叫び系をちゃむ。トーンに調整したもの。

| フック例文 | 使えるシーン |
|-----------|------------|
| え、待って | 驚き・発見系の入り口 |
| ちょっと待って | 注意喚起・反論系 |
| うわ… | 衝撃・想定外の発見 |
| これはマジな話なんだけど | 本気度を伝える入り口 |
| 正直に言うね | 告白・本音系（すでにあるが感情系としても使える） |

叫び系（「ぎゃーーー」「ｱｶﾝｱｶﾝ」）はちゃむ。トーンに合わないため禁止。感嘆符は2個まで。

## コメント欄への引き込み技（oji_to_moji式・全投稿共通）

本文の末尾を「途中」で切るとコメント欄の滞在時間が伸び、スレッズがバズと認識しやすくなる。

**使える文末の切り方**
- 「ちなみに、」で切る → コメント欄に有益情報を続ける
- 「そして、」「あと、」で切る → 補足や体験談をコメント欄へ
- 「続きはコメ欄で」とは書かない。文が途中で終わってる形にする

**例**
本文末尾：「3人育てて結論出たんだけど、ちなみに」
→ コメント欄：「上の子の時に知りたかったやつをまとめると…」

この技は通常投稿・アフィリ投稿どちらでも使える。

## H. 感情スタートフック集（参考：puko.afi01）

> 「商品からじゃなく感情から始めること」がコツ。どの型も「感情の揺らぎ→続きが気になる」構造になっている。

| フック例文 | 感情・効果 |
|-----------|----------|
| 白状します。 | 告白で距離を縮める |
| 9割の人が間違えてるんだけど、 | 「自分は？」と気にさせる |
| 誰も言わないから言うけど、 | 禁断情報感・希少性 |
| 〇〇って人、ちょっと待った | ターゲット指定×引き止め |
| まっっっじで人生No.1の… | 誇張×体験の強調 |
| これ〇〇？！って毎回聞かれる | 実績・問われた経験 |
| 〜とかいう天才アイテム見つけた | 発見・興奮感 |
| 今日やらかしました | 自己開示・親近感 |
| 知らないと損します。 | 損失回避の訴求 |
| ぶっちゃけ、 | 本音・告白の入り口 |

**ちゃむ。への適用メモ**
- 「知らないと損します」は煽りが強いため頻用禁止。月1本まで
- 「今日やらかしました」は保育士×母の失敗告白と相性が良い
- 「ぶっちゃけ、」は `03_writing.md` の本音フレームと組み合わせると効く

# NGフック集（沈むパターン・使用禁止）

- **❶❷❸リスト＋仕組み解説**：「イヤイヤ期の仕組みを解説します❶〜」→ 教科書感で沈む
- **「〇〇になる3つの理由」「〇〇の仕組み」型タイトル**：ブログ記事の見出し臭。1行目には絶対使わない
- **型2（敗因報告）の連投**：単発ならOKだが連投すると暗いアカウントに見える。間隔を空ける
- **「〜ましょう」「〜ではないでしょうか」**：ちゃむ。の口語ではない。全面禁止
- **「人生損してる」「全財産使っても」「革命起きてる」**：大げさすぎてAI感・広告感が出る

# 1行目セルフチェック

投稿前に1行目だけ読んで確認する。

1. 場面か言葉か数字、どれか具体的なものが入ってるか
2. ブログのタイトルっぽくないか（「〜の理由」「〜の仕組み」になってないか）
3. 声に出して読んで、友達へのLINEとして自然か
4. 「今日から使える」or「罪悪感が軽くなる」or「わかる」のどれかを約束できてるか
5. 直近の投稿と同じ型・同じ書き出しになってないか
