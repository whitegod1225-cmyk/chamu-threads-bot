# エンゲージメント・信頼構築型（商品なし）

アフィリエイトリンクを含まず、フォロワーとの信頼・共感を積み上げる投稿の型。
バズ狙いよりも「この人をフォローし続けたい」と思わせることが目的。

---

## 型E-1：炎上覚悟共感型

**元ネタ**：ebi._.ikuji の「ママが満たされていることが大事だよ」投稿

### 構造

```
[フック] 炎上覚悟だけど、〇〇が大事だよ。（覚悟宣言で注意を引く）
[現状肯定リスト] 「〇〇しよう」「〇〇してあげよう」（読者がすでにやっていることを列挙）
---（スライド2）---
[反問の畳み掛け] 〇〇は誰がしてくれるの？ × 3回（読者自身の不在を気づかせる）
[比喩で概念化] 愛情ってコップに注がれた水みたいなもので…（視覚的な比喩）
[真実の提示] なのに〇〇を満たす方法はほとんど語られてない
[解決提案] 〇〇を作ることに気を向けてみて（具体的・小さいアクション）
```

### ポイント

- 「炎上覚悟だけど」は注意喚起であり、「でも言わなきゃいけない」という誠実さの表明
- スライド1で読者の「頑張り」を全力で認めてから転換する（批判ではなく寄り添い）
- 反問の畳み掛けは同じ構造を3回繰り返す（テンポが大事）
- 比喩（コップの水）は抽象的な感情を誰でも分かる言葉に落とし込む
- 解決提案は「誰かに頼れ」ではなく「自分でできる小さいこと」にする

### ちゃむ。への適用

- フック例：「炎上覚悟だけど、保育士から見て正直に言います」「言いにくいけど、現場では当たり前のことを言う」
- 反問例：「子どもの感情は誰が守るの？→ じゃあ、あなたの感情は？」
- 保育士16年の現場知識が「炎上覚悟」の重みと説得力になる

## 型E-2：専門家問題再定義型

**元ネタ**：ebi._.ikuji の「ママの怒りが抑えられないのはSOSサインです」投稿

### 構造

```
[フック] 危険です。〇〇は〇〇のSOSサインです。（断言と危機感）
[状況の描写] 〇〇、〇〇、〇〇…（読者の日常を列挙）
[真の原因の提示] 単に〇〇の話じゃなくて、【〇〇が問題】（一般論を否定して本質を出す）
[科学・専門知識の裏付け] 人は〇〇がないと〇〇する力が落ちる
---（スライド2）---
[「必要なのはこれじゃない」の転換] 「〇〇を頑張ろう」という話ではなくて
[本当の解決] 「〇〇を減らす」を考えること
[具体的行動] たとえば、〇〇する・〇〇でいいことにする（小さい3つ）
[締め] そういった〇〇のサインを見逃さないでね
```

### ポイント

- 「危険です」「SOSサインです」は専門家だから言える断言フレーム
- 「これじゃない→本当にこれ」という二段構えの転換が信頼構築の核心
- 解決策は「頑張り方を変えるのではなく、頑張る量を減らす」のように既存の努力を否定しない
- 具体的行動は小さく・すぐできるものを3つ以内で出す（ハードル最低）

### ちゃむ。への適用

- フック例：「これ、ただの甘えじゃないよ。心のSOSサインだよ」「保育士から見て、これは危険信号です」
- 専門知識の出し方：「16年現場にいてわかったんだけど、〇〇の問題は実は〇〇にある」
- 転換フレーム：「必要なのは〇〇じゃなくて、〇〇を見直すこと」

## 型E-3：著名人文脈乗り×科学的裏付け型

**元ネタ**：ebi._.ikuji の「コール&レスポンス育児」投稿

### 構造

```
[フック] 〇〇や〇〇が使ってるって話してた「〇〇」が楽しすぎる（著名人の話題に乗る）
[具体例] こんな感じで → [会話形式で実演]
[科学的裏付け] 実は〇〇的にめちゃくちゃ理にかなってて
---（スライド2）---
[仕組みの説明] 脳内では❶〜❸（番号付きで視覚的に）
[専門用語を平易に] 〇〇って、〇〇のことで（定義を一言で添える）
[既存方法との比較優位] 「〇〇！」と叱られるより〜（従来との違い）
```

### ポイント

- 著名人（ミキティ・横澤夏子など）の名前を出すことで「話題の情報」として権威付け
- 会話形式（ママ：「〇〇？」→ 子ども：「〇〇！」）で読者がすぐ試せるイメージを作る
- 科学用語（ワーキングメモリ・前頭前野）を出した直後に平易な説明を添える
- 「叱るより〇〇の方が良い」という比較で既存の苦労を肯定しながら転換する

### ちゃむ。への適用

- 著名人の代わりに「フォロワーさんから教えてもらった」「支援センターで先生が話してた」でも機能する
- 発達・感情調整・ワーキングメモリなど保育士知識と相性◎
- 「これって実は〇〇的に理にかなってる」フレームはちゃむ。の発信に自然に使える

## 型E-4：バズ報告エンゲージメント維持型

**元ネタ**：ebi._.ikuji の「え、まってまって、一個前の投稿の反応が凄いね！」投稿

### 構造

```
[驚き報告] え、まってまって、前の投稿の反応が凄いね！
[読者の状態への共感] みんな〇〇なのね。毎日本当に頑張ってるね、お疲れさま。
[ねぎらい] ありがとう🫶
[小さい提案] 〇〇したら、自分のための時間も作ろうね
```

### ポイント

- 単体投稿ではなく「前の投稿がバズったとき」専用の型
- 数字・いいね数ではなく「反応」という言葉でコミュニティ感を演出する
- 読者の行動（寝かしつけ・家事）を想像して声をかけることで親密感を作る
- 投稿の長さは短くていい（2〜3行）。余韻を壊さず共感を延長する役割

### ちゃむ。への適用

- 「前の投稿、思ったより反応もらえて嬉しかった」
- 「こんなに共感してくれる人がいたんだって気づいた」
- バズ直後に必ず出すことで「読んでくれてありがとう」という誠実さが伝わる

### バリエーション：イベント問いかけエンゲージメント型

**元ネタ**：muji_ota の「楽天スーパーセール、何買う？」

セール・イベント期間中に読者に問いかけてコメントを引き出す。商品紹介なし。

```
[問いかけ] 〇〇（イベント名）、何買う？
[自分の本音でオチ] 〇〇ありすぎて、ま、いっか。ってなりそう。
```

> セール直前に出すと「何が良い？」コメントが集まり次の投稿ネタになる。
> 2〜3行で完結させる。長く書かない。
> ちゃむ。例：「楽天セール、何買うか決めた？わたしは悩みすぎて全部カートに入れてる件」

## 訴求の本質：「未来を売る」4ステップ（モジル式）

> 出典：モジル note「アフィリエイトを超簡単に攻略しちゃう方法」
> **人は商品の中身（機能・スペック）では動かない。「これを使ったら自分は変われる」と未来がはっきり見えた時に財布を開く。**

### 売ってはいけないもの vs 売るべきもの

| ❌ 売ってはいけない | ✅ 売るべき |
|-----------------|----------|
| 成分・機能・スペック | 使った後の未来・ゴール |
| 「〇〇が入ってます」 | 「〇〇が手に入る」 |
| 商品の仕様説明 | 読者の不満が消える場面 |

> おにぎりの例：「新潟産の米で十勝の鮭使用」← お腹が空いてる人には不要。
> 「噛んだ瞬間に塩分と香りが口を満たす」← これが"未来の体験"を売っている。

---

### 4ステップ：未来を売る本文構成

```
（1）読者の〈今〉を言葉にする
　　→ 今の不満・コンプレックス・諦め・繰り返してる失敗

（2）〈理想の未来〉をありありと描く
　　→ 「こうなれたら最高」という姿。具体的な場面・感情で

（3）〈差分〉を感じさせる
　　→ 「今のまま」と「こうなれる」のギャップを読者に自覚させる

（4）その橋渡しが商品だと伝える
　　→ 「このギャップを埋めるのがコレ」
```

**商品は〈主役〉ではなく、〈過去から未来へ渡る橋〉。**
この順番に変えるだけで、同じ商品でも成約率がガラッと変わる。

---

### ちゃむ。への適用例

| 商品 | ❌ 機能説明 | ✅ 未来訴求 |
|-----|-----------|----------|
| 防臭袋 | 3層フィルムで臭いを遮断 | 玄関に出してもにおわない。ゴミの日まで気にならない |
| バランスドーム | 体幹トレーニングができる | 外遊びできない日でも、子どもが自分で発散してくれる |
| 知育おもちゃ | 木製・知育効果あり | 「ねえ見て見て」ってくるから、なんか一緒にいる時間が増えた |

---

### 組み合わせるべき型

- S-5（逆説後悔型）：変化の連鎖が「未来の描写」になる
- S-14（自己悩み開示→解決報告型）：ステップ(1)→(4)をそのまま投稿構造にできる
- E-1（炎上覚悟共感型）：読者の〈今〉の肯定→本当に必要なものへの転換

## 構造横断メモ

- **スライド分割の原則**：フック＋文脈をスライド1、商品情報・本体・URLをスライド2以降に置く。スライド1で「続きが気になる」状態を作ってから読ませる
- **URL配置**：コメント欄 or リスト内埋め込み。独立した「↓こちら↓」行はちゃむ。トーンでは広告感が出やすい
- **締め方の種類**：①実績で約束（「ほぼかからない」）②感情行動報告（「速攻調べた」）③季節タイミング（「今年の梅雨に活躍してもらいます」）④逆転発想（「いっそ主役にするのが正解かも」）
- **スライド数の目安**：2枚が基本。S-6・S-7など情報量が多い場合は3〜5枚まで。1枚に詰め込まない

## 投稿制作の3原則（sukimarisu式）

> 出典：sukimarisu2415（Threads）

---

### 原則1：数字を1つ入れる（「ふーん」→「え、それほしい」）

抽象的な言葉を具体的な数字に置き換えるだけで、読者の反応が変わる。

| ❌ 抽象 | ✅ 数字あり |
|--------|----------|
| 時短グッズおすすめ | 朝の準備が15分→8分になったグッズ |
| 寝かしつけにいいよ | 寝かしつけ45分が10分になった |
| 離乳食ラクになった | 日曜30分仕込むだけで平日全部レンチン |

> ちゃむ。への適用：保育士16年・3人子育ての経験は「自分の数字」に変換できる。
> 「子ども3人試した」「6年使い続けてる」「夜泣きが3日で落ち着いた」など体験値を数字で出す。

---

### 原則2：ピークエンドの法則（1行目と最後だけ本気で書く）

人の記憶に残るのは「いちばん盛り上がった瞬間」と「最後の印象」だけ。途中は7割でいい。

```
1行目 → 「あるある」か「えっ？」で止める（保存・続きを読む動機を作る）
最後  → 「同じ人いる？」など未完了感を残す（コメントが来る・記憶に残る）
真ん中 → 7割の完成度で出す。完璧にしなくていい
```

> ちゃむ。への適用：
> - フックはすでに型で押さえてある（12_hook-patterns.md）
> - 締めの一文に「同じ人いる？」「これあるある？」を意識的に入れる
> - 真ん中に時間をかけすぎない。フック→要点→締めの順で構成する

---

### 原則3：ネタは「昨日何があった？」で考える

「何を書こう」という発想を捨てる。育児してる時点でネタは毎日自動生成される。

```
離乳食こぼされた → ネタになる
寝かしつけ45分かかった → ネタになる
子どもが靴を左右逆に履いてた → ネタになる
```

「書くことがない」のではなく「書いていいと思っていない」だけ。

> ちゃむ。への適用：
> - 「昨日の育児エピソード」を入り口にする型は S-9（笑えるエピソード）・S-2（日常シーン報告）がある
> - エピソードをメモしておき、型に当てはめるだけで投稿になる
> - 「3児の母×元保育士」の日常は読者にとってコンテンツそのもの
